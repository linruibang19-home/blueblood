# 14 · 核心链路四：钱包提现

> 目标：理解钱包余额、提现申请、管理员审核、驳回退款的资金流。

---

## 一、钱包模块做什么？

钱包模块负责：

```text
1. 查看账户余额
2. 查看收益流水
3. 任务结算入账
4. 发起提现申请
5. 管理员审核提现
6. 驳回提现时退回余额
```

核心表：

```text
wallet_account     钱包账户余额
wallet_record      钱包流水
withdraw_record    提现申请
```

---

## 二、提现链路总览

```text
用户发起提现
→ 校验余额是否足够
→ 原子扣减 wallet_account.balance
→ 创建 withdraw_record，状态 PENDING
→ 创建 wallet_record，状态 withdrawing
→ 管理员后台审核
   ├── APPROVED：标记已打款
   └── REJECTED：余额加回，提现失败
```

---

## 三、为什么提现时先扣余额？

假设用户余额 1000，同时发起两笔提现：

```text
提现 A：800
提现 B：800
```

如果不先扣余额，就可能两笔都通过，最后平台需要打款 1600，但用户其实只有 1000。

所以提现申请时必须：

```text
先冻结/扣减余额
```

项目中用的是：

```sql
UPDATE wallet_account
SET balance = balance - ?
WHERE user_id = ? AND balance >= ?
```

如果影响行数是 0，说明余额不足。

---

## 四、用户发起提现

用户操作：

```text
我的 → 钱包 → 申请提现 → 输入金额 → 提交
```

后端链路：

```text
POST /wallet/withdraw
→ WalletService.createWithdraw(req)
→ 当前用户 ID = SecurityUtils.currentUserId()
→ WalletAccountMapper.deductBalance(userId, amount)
→ 创建 withdraw_record(status=PENDING)
→ 创建 wallet_record(type=withdraw, status=withdrawing)
```

数据变化：

```text
wallet_account.balance 减少
withdraw_record 新增一条 PENDING 记录
wallet_record 新增一条提现流水
```

---

## 五、管理员审核提现

管理员操作：

```text
后台管理 → 财务管理 → 提现管理 → 审核
```

后端链路：

```text
POST /admin/wallet/withdraw/{id}/review
→ AdminWalletService.reviewWithdraw(id, req)
→ 查询 withdraw_record
→ 判断当前状态是否 PENDING
→ 根据审核结果处理
```

### APPROVED

通过审核：

```text
withdraw_record.status = APPROVED
wallet_record.status = withdrawn
记录 reviewer_id / review_time
```

含义：

```text
平台确认已经打款，余额已经在申请时扣掉，不需要再扣。
```

### REJECTED

驳回审核：

```text
withdraw_record.status = REJECTED
wallet_record.status = rejected
wallet_account.balance += amount
```

含义：

```text
提现失败，之前冻结/扣掉的余额必须退回。
```

---

## 六、为什么驳回必须退款？

这是一个资损风险点。

如果提现申请时已经扣了余额，但驳回时不退回：

```text
用户余额减少了
平台又没有打款
用户的钱凭空消失
```

这就是资损 bug。

正确做法：

```text
REJECTED 分支必须 addBalance 回补余额。
```

---

## 七、钱包流水怎么理解？

`wallet_record` 记录每一笔资金变化。

常见类型：

```text
income      收入，例如任务里程碑结算
withdraw    提现，例如用户申请提现
```

常见状态：

```text
available     可用收入
withdrawing   提现中
withdrawn     已提现
rejected      提现驳回
```

作用：

```text
wallet_account 只存当前余额；
wallet_record 记录历史变化。
```

## 八、钱包模块的并发安全

### 入账

```sql
UPDATE wallet_account
SET balance = balance + ?, total_earned = total_earned + ?
WHERE user_id = ?
```

防止并发入账丢失更新。

### 提现

```sql
UPDATE wallet_account
SET balance = balance - ?
WHERE user_id = ? AND balance >= ?
```

防止余额变成负数。

## 九、这条链路怎么排错？

### 提现提示余额不足

可能原因：

```text
wallet_account.balance < amount
提现金额非法
前端展示余额不是最新
```

### 提现申请成功但后台看不到

可能原因：

```text
withdraw_record 没插入
后台筛选状态不对
admin 接口权限失败
```

### 驳回后余额没回来

可能原因：

```text
REJECTED 分支没执行 addBalance
事务回滚
wallet_account 缺失
```

### 重复审核

需要检查：

```text
审核前是否判断 status=PENDING
已审核记录是否禁止重复审核
```
