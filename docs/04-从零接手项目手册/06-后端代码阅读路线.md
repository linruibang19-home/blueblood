# 06 · 后端代码阅读路线

> 目标：知道 Spring Boot 后端应该按什么顺序读，不被大量模块吓住。

---

## 一、后端整体结构

后端目录：

```text
server/blueblood-api/src/main/java/com/blueblood/api/
├── BluebloodApiApplication.java
├── common/
├── config/
├── security/
└── modules/
```

不要从 `modules` 随便点开一个类开始看。正确顺序是：

```text
启动类 → 通用规范 → 安全认证 → 核心业务模块 → 后台管理模块
```

---

## 二、第一步：看启动类

文件：

```text
server/blueblood-api/src/main/java/com/blueblood/api/BluebloodApiApplication.java
```

你需要知道：

```text
这是 Spring Boot 应用入口。
运行 main 方法后，Spring Boot 会扫描 com.blueblood.api 包下的组件。
```

---

## 三、第二步：看通用规范 common/

先看这些文件：

```text
common/result/Result.java
common/result/ResultCode.java
common/result/PageResult.java
common/exception/BusinessException.java
common/exception/GlobalExceptionHandler.java
```

### 你要理解什么？

#### 统一响应

项目所有接口基本返回：

```json
{
  "code": 0,
  "message": "成功",
  "data": {}
}
```

这样前端只需要判断：

```text
code == 0  成功
code != 0  失败
```

#### 全局异常

Service 层遇到业务错误时抛：

```java
throw new BusinessException(ResultCode.OPERATION_FAILED, "名额已满");
```

然后 `GlobalExceptionHandler` 统一转成 JSON。

## 四、第三步：看配置 config/

重点文件：

```text
config/SecurityConfig.java
config/RedisConfig.java
config/MybatisPlusConfig.java
config/Knife4jConfig.java
```

### SecurityConfig

你要看：

```text
1. 哪些接口 permitAll
2. 哪些接口需要登录
3. JWT 过滤器放在哪个位置
4. 401/403 如何返回 JSON
```

### RedisConfig

你要看：

```text
1. RedisTemplate 怎么序列化
2. StringRedisTemplate 用在哪些地方
```

### MybatisPlusConfig

你要看：

```text
分页插件是否配置
自动填充是否配置
```

## 五、第四步：看认证 security/

重点文件：

```text
security/JwtUtil.java
security/JwtAuthenticationFilter.java
security/SecurityUtils.java
security/LoginUser.java
security/RestAuthenticationEntryPoint.java
security/RestAccessDeniedHandler.java
```

阅读顺序：

```text
1. JwtUtil.generate()         了解 token 怎么生成
2. JwtUtil.isValid()          了解 token 怎么校验
3. JwtAuthenticationFilter    了解请求怎么解析 token
4. SecurityUtils              了解业务代码怎么取当前用户
```

你要建立这条链路：

```text
登录成功 → JwtUtil 生成 token
后续请求 → JwtAuthenticationFilter 解析 token
解析成功 → SecurityContext 保存 LoginUser
Service 中 → SecurityUtils.currentUserId() 获取用户 ID
```

---

## 六、第五步：看业务模块 modules/

每个模块一般结构类似：

```text
modules/xxx/
├── controller/      # 接口入口
├── service/         # 业务逻辑
├── mapper/          # 数据库访问
├── entity/          # 表映射
└── dto/             # 请求/响应对象
```

阅读每个模块时按这个顺序：

```text
Controller → Service → Mapper → Entity → DTO
```

不要先看 Entity，不然容易被字段淹没。

---

## 七、核心模块阅读顺序

### 1. auth：登录注册

看：

```text
modules/auth/controller/AuthController.java
modules/auth/service/AuthService.java
modules/auth/service/VerificationCodeService.java
```

要理解：

```text
登录怎么校验密码？
验证码怎么存 Redis？
JWT 怎么签发？
```

### 2. user：用户资料与角色

看：

```text
modules/user/
```

要理解：

```text
User 是登录主体
UserProfile 是扩展资料
SysRole / SysUserRole 是角色权限
UserVerification 是实名认证
```

### 3. task：项目最核心模块

重点看：

```text
modules/task/controller/TaskController.java
modules/task/service/TaskService.java
modules/task/service/TaskPublishService.java
modules/task/service/MilestoneService.java
modules/task/service/TaskStockService.java
modules/task/service/TaskListCacheService.java
modules/task/mapper/TaskMapper.java
```

阅读顺序：

```text
1. TaskController：有哪些接口
2. TaskService.pageTasks：任务大厅怎么查
3. TaskService.detail：任务详情怎么查
4. TaskService.accept：接单核心逻辑
5. TaskPublishService.publish：发布任务
6. MilestoneService.submit：提交里程碑
7. MilestoneService.review：审核里程碑
8. TaskStockService：Redis 秒杀库存
9. TaskListCacheService：任务列表缓存
```

### 4. wallet：钱包结算

看：

```text
modules/wallet/service/WalletService.java
modules/wallet/mapper/WalletAccountMapper.java
modules/wallet/mapper/WalletRecordMapper.java
modules/wallet/mapper/WithdrawRecordMapper.java
```

要理解：

```text
credit 入账怎么幂等？
提现为什么先扣余额？
驳回提现为什么要加回余额？
```

### 5. enterprise：企业认证与机会发布

看：

```text
modules/enterprise/service/EnterpriseService.java
modules/enterprise/entity/EnterpriseApplication.java
modules/enterprise/entity/Job.java
modules/enterprise/entity/Hackathon.java
```

要理解：

```text
个人用户如何变成企业用户？
企业用户为什么才能发布任务/岗位/黑客松？
```

### 6. admin：后台管理

看：

```text
modules/admin/controller/
modules/admin/service/
```

要理解：

```text
后台接口一般是 /admin/*
多数接口需要 @PreAuthorize("hasRole('ADMIN')")
后台做的是运营管理和兜底审核
```

## 八、接单链路代码阅读路线

如果只想看最核心链路，按这个顺序：

```text
1. TaskController.accept()
2. TaskService.accept()
3. TaskStockService.tryDeduct()
4. TaskOrderMapper.insert()
5. TaskMilestoneMapper.insert()
6. TaskMapper.decrementSlotsLeft()
7. TaskStockService.rollback()
```

你要回答：

```text
Redis 为什么先扣？
DB 为什么还要扣？
失败为什么要回补 Redis？
重复接单怎么兜底？
里程碑模板怎么变成订单实例？
```
