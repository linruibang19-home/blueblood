# 17 · 常见问题与排坑清单

> 目标：遇到启动、登录、权限、数据库、前后端联调问题时，先看这里。

---

## 一、启动类问题

### 1. Docker 启动失败

检查：

```text
[ ] Docker Desktop 是否启动
[ ] 端口是否被占用
[ ] 后端 jar 是否已打包
[ ] docker-compose 文件路径是否正确
```

推荐命令：

```bash
docker compose -f deploy/docker-compose.full.yml ps
```

查看日志：

```bash
docker compose -f deploy/docker-compose.full.yml logs api
```

---

### 2. 后端容器启动失败

常见原因：

```text
target/*.jar 不存在
MySQL 连接失败
Redis 连接失败
环境变量不正确
```

先执行：

```bash
cd server/blueblood-api
mvn -q clean package -DskipTests
cd ../..
docker compose -f deploy/docker-compose.full.yml up -d --build
```

---

### 3. MySQL 初始化不对

如果数据库数据很乱，想重来：

```bash
docker compose -f deploy/docker-compose.full.yml down -v
docker compose -f deploy/docker-compose.full.yml up -d --build
```

注意：

```text
-v 会删除数据库卷，数据会清空。
```

---

## 二、端口问题

项目常用端口：

| 端口 | 服务 |
|---|---|
| 3000 | 前台 H5 |
| 3001 | 后台管理 |
| 8090 | 后端 API |
| 3307 | MySQL |
| 6380 | Redis |

如果访问失败，先确认对应服务是否运行。

## 三、登录问题

### 1. 登录失败

检查：

```text
[ ] 账号密码是否正确
[ ] 后端 API 是否启动
[ ] 前端是否请求到正确 baseURL
[ ] 数据库是否有种子账号
```

默认账号：

```text
lin / 123456
admin / 123456
```

### 2. 登录成功但后续请求 401

可能原因：

```text
token 没保存
token 没带到 Authorization header
Authorization 格式不是 Bearer xxx
token 过期
token 被清理
```

重点看：

```text
前端请求拦截器
后端 JwtAuthenticationFilter
```

### 3. 后台登录后 403

403 表示：

```text
你已经登录，但权限不够。
```

原因：

```text
不是 ADMIN 角色
@PreAuthorize("hasRole('ADMIN')") 校验失败
```

## 四、前端数据问题

### 1. 前台看不到真实数据

检查是不是 Mock 模式。

如果要连后端：

```bash
cd apps/user-web
npm run dev:api
```

不要用：

```bash
npm run dev
```

除非你只想看 Mock 页面。

### 2. 请求路径不对

确认：

```text
后端 API 前缀是 /api
前端代理是否把 /api 转发到 localhost:8090
```

---

## 五、任务接单问题

### 1. 提示名额已满

检查：

```text
Redis task:slots:{taskId}
MySQL task.slots_left
任务状态是否允许接单
```

### 2. 提示已接取该任务

原因：

```text
当前用户已有 task_order
唯一索引兜底触发
```

### 3. 接单成功但跳转失败

检查：

```text
后端是否返回 orderId
前端是否正确读取 data.orderId
路由参数是否正确
```

---

## 六、钱包问题

### 1. 里程碑通过但没入账

检查：

```text
milestone 是否 APPROVED
WalletService.credit 是否执行
wallet_record 是否已有相同 biz_type/biz_id
wallet_account 是否存在
事务是否回滚
```

### 2. 提现驳回后余额没回来

检查：

```text
AdminWalletService.reviewWithdraw 的 REJECTED 分支
是否调用 WalletAccountMapper.addBalance
事务是否提交
```

---

## 七、Redis 问题

### 1. Redis 数据丢了怎么办？

部分 key 可以恢复：

```text
task:slots:* 可以从 DB slots_left 懒加载
验证码丢了重新发送
缓存丢了重新查 DB
幂等 token 丢了可能需要重新打开页面获取
```

### 2. Redis 连不上

检查：

```text
Docker redis 容器是否运行
端口是否是 6380（宿主机）/6379（容器内）
后端配置的 host 是否正确
```

---

## 八、数据库问题

### 1. 查询不到数据

检查：

```text
是否连的是 blueblood_v2
是否导入了 init.sql
是否被 deleted_at 软删过滤
查询条件是否正确
```

### 2. 新增数据看不到

检查：

```text
是否事务未提交
是否前端走的是 Mock 数据
是否查询时过滤了 status/deleted_at
```

---

## 九、排坑口诀

遇到问题按这个顺序查：

```text
1. 服务有没有启动？
2. 请求有没有发出去？
3. token 有没有带？
4. 后端有没有收到？
5. 权限有没有通过？
6. 数据库有没有数据？
7. Redis 有没有对应 key？
8. 前端有没有正确解包？
```

不要一上来改代码，先定位是哪一层的问题。
