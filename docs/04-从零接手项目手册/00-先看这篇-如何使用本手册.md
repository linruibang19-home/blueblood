# 00 · 先看这篇：如何使用本手册

> 你现在的目标不是立刻背八股，也不是立刻改代码，而是先恢复对项目的整体掌控感。

---

## 一、先明确：你为什么会“忘记项目”？

这是正常的。这个项目包含：

- 4 端：前台 H5、后台管理、小程序、后端 API；
- 业务域很多：社区、课程、任务、钱包、企业、通知、系统；
- 技术点很多：JWT、RBAC、MySQL、Redis、Docker、缓存、限流、幂等；
- 文档也很多：开发方案、交付成果、面试准备。

所以几天不看后，大脑会先丢失“整体地图”。这时候不要直接打开代码乱翻，而要按顺序恢复。

---

## 二、正确恢复顺序

推荐顺序是：

```text
1. 先恢复项目定位
2. 再理解目录结构
3. 再跑通项目
4. 再从用户视角走业务流程
5. 再读后端核心代码
6. 再理解数据库和 Redis
7. 最后做自测题
```

不要一开始就看 `TaskService.accept()`，也不要一开始就看 44 张表。那样容易越看越乱。

## 六、每篇文档怎么看？

建议每篇都按这个方法：

```text
1. 先看标题和目录，知道它要解决什么问题
2. 再看流程图或表格，建立整体感
3. 再打开对应代码文件对照看
4. 最后合上文档，用自己的话讲一遍
```

如果你能做到“合上文档讲出来”，才算真的恢复了。

---

## 七、不要做的事

不要这样做：

- 一上来直接看所有 Java 文件；
- 一上来直接看 SQL 全表；
- 一上来就背面试答案；
- 一边忘一边焦虑，越翻越乱；
- 只看文档不跑项目。

正确做法是：**先地图，再路线，再细节。**

---

## 八、最终标准

看完这个手册后，你至少应该能回答：

1. 这个项目是干什么的？
2. 四端分别在哪个目录？
3. 后端核心模块在哪里？
4. 如何启动项目？
5. 登录认证怎么走？
6. 任务发布和接单怎么走？
7. Redis 在项目里做了哪些事？
8. 钱包结算为什么要幂等？
9. 如果要加一个新功能，应该改哪些层？

如果这些都能答出来，你就已经重新接手成功了。
