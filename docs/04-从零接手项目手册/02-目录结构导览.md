# 02 · 目录结构导览

> 目标：打开仓库后知道每个目录是干什么的，先看哪里，后看哪里。

---

## 顶层目录总览

```text
bluebloodv2/
├── apps/                  # 三个前端应用：前台 H5、后台管理、小程序
├── server/                # Spring Boot 后端 API
├── sql/                   # 数据库初始化和增量迁移 SQL
├── deploy/                # Docker Compose 部署文件
├── docs/                  # 项目文档
├── prototype/             # 原型 HTML
├── README.md              # 项目总入口说明
└── .gitignore
```

## 你应该先看哪些？

如果你刚接手项目，建议顺序是：

```text
1. README.md
2. docs/04-从零接手项目手册/
3. deploy/docker-compose.full.yml
4. server/blueblood-api/
5. apps/user-web/
6. apps/admin-web/
7. sql/init.sql
8. apps/miniapp/
```

不要一上来就看 `node_modules`、`dist` 或所有 SQL。

## apps/：三个前端端

```text
apps/
├── user-web/       # 前台 H5，用户主要入口
├── admin-web/      # 后台管理端，管理员入口
└── miniapp/        # 微信小程序，移动端轻量入口
```

### apps/user-web

定位：前台 H5。

用户在这里完成：

- 登录注册；
- 浏览发现/社区；
- 学习课程；
- 浏览任务大厅；
- 接单、发布任务、提交里程碑；
- 查看钱包、通知、个人中心。

重点目录：

```text
apps/user-web/src/
├── pages/          # 页面
├── components/     # 通用组件
├── layouts/        # 移动端布局
├── router/         # 路由
├── stores/         # Pinia 状态
├── api/            # 请求封装
└── mock/           # Mock 数据
```

### apps/admin-web

定位：后台管理端。

管理员在这里完成：

- 用户管理；
- 企业认证审核；
- 任务管理；
- 接单/里程碑审核；
- 钱包提现审核；
- 系统配置、通知、日志。

重点目录：

```text
apps/admin-web/src/
├── views/          # 后台页面
├── layout/         # 后台布局
├── router/         # 路由
├── stores/         # 状态
└── api/            # 请求封装
```

### apps/miniapp

定位：微信小程序。

用于移动端轻量操作：

- 任务大厅；
- 任务详情；
- 我的订单；
- 提交里程碑；
- 钱包；
- 通知。

重点目录：

```text
apps/miniapp/src/
├── pages/          # 小程序页面
├── stores/         # 状态
├── utils/          # 请求封装等工具
└── App.vue         # 应用入口
```

## server/：后端 API

```text
server/
└── blueblood-api/
    ├── pom.xml
    ├── Dockerfile
    └── src/main/
        ├── java/com/blueblood/api/
        └── resources/
```

后端是项目最核心部分。

重点目录：

```text
server/blueblood-api/src/main/java/com/blueblood/api/
├── BluebloodApiApplication.java       # Spring Boot 启动类
├── common/                            # 通用结果、异常、分页、限流、幂等
├── config/                            # Security、Redis、MyBatis、Knife4j 配置
├── security/                          # JWT、登录用户、安全工具
└── modules/                           # 业务模块
```

### common/

通用基础设施：

```text
common/
├── result/            # Result、PageResult、ResultCode
├── exception/         # BusinessException、GlobalExceptionHandler
├── ratelimit/         # @RateLimit + RateLimitAspect
└── idempotent/        # IdempotentService
```

新手先看：

- `Result.java`
- `ResultCode.java`
- `GlobalExceptionHandler.java`
- `RateLimitAspect.java`
- `IdempotentService.java`

### config/

系统配置：

```text
config/
├── SecurityConfig.java       # Spring Security 配置
├── RedisConfig.java          # Redis 配置
├── MybatisPlusConfig.java    # MyBatis-Plus 分页等配置
├── Knife4jConfig.java        # 接口文档配置
└── WebMvcConfig.java         # Web MVC 配置
```

新手重点看：

- `SecurityConfig.java`
- `RedisConfig.java`

### security/

认证鉴权：

```text
security/
├── JwtAuthenticationFilter.java
├── JwtUtil.java
├── JwtProperties.java
├── LoginUser.java
├── SecurityUtils.java
├── RestAuthenticationEntryPoint.java
└── RestAccessDeniedHandler.java
```

新手重点看：

- `JwtAuthenticationFilter.java`
- `JwtUtil.java`
- `SecurityUtils.java`

### modules/

业务模块：

```text
modules/
├── auth/              # 登录、注册、验证码、wx-login
├── user/              # 用户、资料、认证、角色
├── community/         # 社区、小组、帖子、评论、聊天
├── course/            # 课程、章节、学习进度
├── assignment/        # 作业提交、评分
├── task/              # 任务、接单、里程碑、秒杀库存
├── wallet/            # 钱包、流水、提现
├── notification/      # 通知、已读
├── system/            # 文件、配置、字典、日志
├── enterprise/        # 企业认证、岗位、黑客松
└── admin/             # 后台管理接口/服务
```

最重要的模块：

```text
auth    登录认证
user    用户和角色
task    任务接单核心
wallet  钱包结算核心
admin   后台管理
```

## sql/：数据库

```text
sql/
├── init.sql
├── v2_001_auth_user.sql
├── v3_001_enterprise.sql
├── v4_001_task_publish_settlement.sql
└── v5_001_course_seed.sql
```

作用：

- `init.sql`：初始化库、表、种子数据；
- `v2~v5`：增量迁移，用于已运行数据库升级。

新手不要一上来逐行读 SQL，应该先看 [10-数据库从零理解](10-数据库从零理解.md)。

## 最容易混淆的几个点

| 容易混淆 | 正确理解 |
|---|---|
| user-web 和 admin-web | 一个是用户端，一个是管理员后台 |
| task 和 order | task 是任务模板/任务大厅项，order 是用户接单后的订单实例 |
| task_milestone | 发布任务时是模板，接单后复制成订单实例 |
| wallet_record 和 withdraw_record | 前者是钱包流水，后者是提现申请 |
| Redis 库存和 DB 库存 | Redis 挡高并发，DB 保最终正确 |
| USER 角色和 enterprise 用户类型 | USER/ADMIN 是角色，personal/enterprise 是用户类型 |
