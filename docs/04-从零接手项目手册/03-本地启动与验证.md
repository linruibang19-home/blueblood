# 03 · 本地启动与验证

> 目标：把项目跑起来，并确认前台、后台、后端、MySQL、Redis 都正常。

---

## 一、推荐启动方式：Docker 五容器

这是最适合重新接手项目的方式，因为不用分别启动 MySQL、Redis、前后端。

五个容器包括：

```text
blueblood-v2-mysql       MySQL 8
blueblood-v2-redis       Redis 7
blueblood-v2-api         Spring Boot API
blueblood-v2-user-web    前台 H5
blueblood-v2-admin-web   后台管理端
```

---

## 二、启动前准备

确认本机有：

```text
[ ] Docker Desktop
[ ] JDK 17
[ ] Maven
[ ] Node.js（如果需要本地跑前端）
```

如果只用 Docker 起完整栈，最关键是 Docker Desktop 正常运行。

---

## 三、Docker 一键启动

在项目根目录执行：

```bash
cd d:/Dev/Intership/blueblood-project/bluebloodv2
```

先打后端 jar：

```bash
cd server/blueblood-api
mvn -q clean package -DskipTests
cd ../..
```

启动完整栈：

```bash
docker compose -f deploy/docker-compose.full.yml up -d --build
```

查看容器状态：

```bash
docker compose -f deploy/docker-compose.full.yml ps
```

正常情况应该看到 5 个服务都是 `Up`。

## 四、启动后访问地址

| 服务 | 地址 | 说明 |
|---|---|---|
| 前台 H5 | http://localhost:3000 | 用户端入口 |
| 后台管理 | http://localhost:3001 | 管理员入口 |
| 后端 API | http://localhost:8090/api | API 根路径 |
| 接口文档 | http://localhost:8090/api/doc.html | Knife4j 文档 |
| MySQL | localhost:3307 | root / blueblood |
| Redis | localhost:6380 | 无密码 |

## 五、快速验证清单

启动后按顺序验证：

```text

```

## 六、本地原生开发启动方式

如果你想改代码并热更新，可以本地原生启动。

### 1. 后端

需要本机 MySQL 和 Redis 可用。

```bash
cd server/blueblood-api
SERVER_PORT=8090 MYSQL_PASSWORD=<你的MySQL密码> mvn spring-boot:run
```

后端默认 API 前缀：

```text
/api
```

### 2. 前台 H5

Mock 模式：

```bash
cd apps/user-web
npm install
npm run dev
```

真实后端模式：

```bash
cd apps/user-web
npm run dev:api
```

区别：

| 命令 | 数据来源 |
|---|---|
| `npm run dev` | Mock 数据 |
| `npm run dev:api` | 真实后端 API |

### 3. 后台管理

```bash
cd apps/admin-web
npm install
npm run dev
```

### 4. 小程序

```bash
cd apps/miniapp
npm install
npm run dev:mp-weixin
```

然后用微信开发者工具打开：

```text
apps/miniapp/dist/dev/mp-weixin
```

## 七、启动失败常见原因

### 1. 端口被占用

常见端口：

```text
3000  前台 H5
3001  后台管理
8090  后端 API
3307  MySQL
6380  Redis
```

如果端口被占用：

```bash
docker compose -f deploy/docker-compose.full.yml down
```

或者关闭占用端口的旧服务。

### 2. 后端 jar 没打包

如果 API 容器启动失败，先确认：

```text
server/blueblood-api/target/*.jar
```

是否存在。

如果不存在，重新执行：

```bash
cd server/blueblood-api
mvn -q clean package -DskipTests
```

### 3. MySQL 没初始化

Docker 第一次启动会自动执行 `sql/init.sql`。

如果之前启动过但数据不对，可以清空卷重建：

```bash
docker compose -f deploy/docker-compose.full.yml down -v
docker compose -f deploy/docker-compose.full.yml up -d --build
```

注意：`-v` 会删除数据库数据。

### 4. 前端看不到真实数据

前台 H5 有 Mock/API 模式。

如果你想连真实后端，需要使用：

```bash
npm run dev:api
```

或者检查环境变量：

```text
VITE_USE_API=true
```

## 八、你应该怎么确认“项目真的跑通”？

至少完成一次：

```text
1. 前台登录 lin / 123456
2. 进入任务大厅
3. 打开任务详情
4. 点击接单
5. 进入我的任务查看订单
6. 后台登录 admin / 123456
7. 查看任务/订单/里程碑相关管理页
```

完成这条路线，说明前端、后端、数据库、Redis、权限基本都通了。
