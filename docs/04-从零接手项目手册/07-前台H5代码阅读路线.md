# 07 · 前台 H5 代码阅读路线

> 目标：看懂 `apps/user-web`，知道用户端页面如何组织、如何调后端。

---

## 一、前台 H5 是什么？

目录：

```text
apps/user-web
```

定位：

```text
用户端 H5，也就是普通用户和企业用户主要使用的前台应用。
```

用户在这里完成：

```text
登录注册、浏览社区、学习课程、接任务、发布任务、提交里程碑、钱包提现、查看通知、编辑个人资料。
```

---

## 二、建议阅读顺序

```text
1. package.json             看项目脚本和依赖
2. src/main.ts              看应用入口
3. src/router/              看路由结构
4. src/stores/              看登录状态和用户状态
5. src/api/                 看接口封装
6. src/layouts/             看移动端布局
7. src/pages/               按业务模块看页面
8. src/components/          最后看通用组件
```

---

## 三、主要目录

```text
apps/user-web/src/
├── main.ts                 # 应用入口
├── App.vue                 # 根组件
├── router/                 # 路由
├── stores/                 # Pinia 状态
├── api/                    # API 请求封装
├── pages/                  # 页面
├── components/             # 通用组件
├── layouts/                # 布局
└── mock/                   # Mock 数据
```

---

## 四、先看 package.json

关注脚本：

```text
npm run dev       Mock 模式
npm run dev:api   真实后端 API 模式
npm run build     构建生产包
```

重点记住：

> 如果你想连 Spring Boot 后端，要用 `npm run dev:api`。

---

## 五、看路由 router/

路由能告诉你有哪些页面。

重点页面大致分为：

```text
pages/auth/        登录、注册、企业注册
pages/discover/    发现、社区、小组、帖子、聊天
pages/grow/        成长、课程、作业、岗位、黑客松
pages/task/        任务大厅、任务详情、接单、发布、里程碑
pages/mine/        我的、钱包、通知、资料、设置
```

读路由时关注：

```text
1. 首页默认跳哪里？
2. 哪些页面需要登录？
3. 任务模块有哪些页面？
4. 企业相关页面在哪里？
```

---

## 六、看 stores/

重点看用户状态：

```text
stores/auth 或 stores/user
```

你要理解：

```text
token 存在哪里？
当前用户信息存在哪里？
退出登录时清哪些数据？
401 时怎么处理？
```

典型逻辑：

```text
登录成功 → 保存 token → 保存 user 信息 → 后续请求自动带 Authorization
退出登录 → 清 token → 清 user → 跳登录页
```

---

## 七、看 api/

API 封装负责：

```text
1. 创建 axios 实例
2. 配置 baseURL
3. 请求拦截器加 token
4. 响应拦截器解包 Result<T>
5. 统一处理 401
```

重点找：

```text
auth API：登录注册
user API：用户信息
course API：课程
community API：社区
任务 API：task、order、milestone
wallet API：钱包
```

## 十、Mock/API 模式怎么理解？

前台支持两种模式：

| 模式 | 命令 | 数据来源 | 适合场景 |
|---|---|---|---|
| Mock | `npm run dev` | 前端 Mock 数据 | 没有后端时预览页面 |
| API | `npm run dev:api` | Spring Boot API | 联调真实业务 |

如果页面数据和数据库不一致，先检查是不是还在 Mock 模式。
