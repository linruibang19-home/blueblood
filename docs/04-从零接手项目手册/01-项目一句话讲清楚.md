# 01 · 项目一句话讲清楚

> 目标：用最短时间恢复“这个项目到底是什么”的整体认识。

---

## 一句话定位

**蓝血菁英平台** 是一个面向 AI 技术人才的成长与任务撮合平台：用户可以学习课程、参与社区、接任务交付成果，并在里程碑验收后获得钱包收益。

更口语化地说：

> 它像一个“AI 人才成长 + 接单赚钱”的平台，把学习、社区、任务、结算串成一个闭环。

---

## 项目解决什么问题？

平台想解决的是：

```text
AI 技术人才如何成长？
AI 技术人才如何获得实战机会？
企业/任务方如何找到合适的人？
接单后的交付和结算如何可信闭环？
```

所以项目设计了四个核心方向：

| 方向 | 解决的问题 |
|---|---|
| 成长学习 | 用户通过课程、章节、作业提升能力 |
| 社区协作 | 用户通过小组、帖子、评论、私信互动 |
| 任务撮合 | 用户接企业/平台发布的任务，提交成果 |
| 收益结算 | 任务按里程碑审核通过后进入钱包 |

---

## 四端分别是什么？

| 端 | 目录 | 主要使用者 | 作用 |
|---|---|---|---|
| 前台 H5 | `apps/user-web` | 普通用户、企业用户 | 登录、浏览课程/任务、接单、发布任务、钱包、个人中心 |
| 后台管理端 | `apps/admin-web` | 管理员 | 用户管理、企业审核、任务管理、里程碑审核、财务审核 |
| 微信小程序 | `apps/miniapp` | 移动端用户 | 轻量任务、钱包、通知入口 |
| 后端 API | `server/blueblood-api` | 所有前端 | 统一业务接口、认证鉴权、数据库/Redis 操作 |

最关键的一点：

> 前台 H5、后台管理、小程序共用一套 Spring Boot 后端 API。

---

## 用户角色有哪些？

| 角色 | 标识 | 能做什么 |
|---|---|---|
| 个人用户 | `user_type=personal` | 学课程、看社区、接任务、提交成果、提现 |
| 企业用户 | `user_type=enterprise` | 个人用户能力 + 发布任务/岗位/黑客松 + 审核里程碑 |
| 管理员 | `role=ADMIN` | 后台管理、审核用户/企业/任务/提现、系统配置 |

简单理解：

```text
个人用户：接单赚钱
企业用户：发布任务并验收
管理员：运营与兜底审核
```

---

## 最核心的业务闭环

平台最核心的业务不是社区，也不是课程，而是：

```text
任务发布 → 用户接单 → 里程碑交付 → 雇主审核 → 钱包入账 → 提现
```

详细拆开：

```text
1. 企业/雇主发布任务
   - 配置任务标题、描述、金额、名额、技能要求
   - 配置多个里程碑，每个里程碑有自己的金额

2. 用户接单
   - 系统生成任务订单
   - 把任务的里程碑模板复制为该订单的里程碑实例
   - 扣减任务剩余名额

3. 接单者提交里程碑
   - 提交成果说明、GitHub 地址、附件等

4. 雇主审核里程碑
   - APPROVED：通过
   - REJECTED：驳回

5. 通过后分阶段结算
   - 每个里程碑通过后，按该里程碑金额入账接单者钱包
   - 钱包流水做幂等，防止重复入账

6. 全部里程碑通过
   - 订单状态变为 settled
```

## 项目技术栈一句话

后端：

```text
Spring Boot 3 + MyBatis-Plus + Spring Security + JWT + MySQL 8 + Redis + Knife4j + Docker
```

前端：

```text
Vue3 + Vite + TypeScript + Pinia + Vue Router + Vant / Element Plus / uni-app
```

---

## 项目最值得记住的技术点

| 技术点 | 项目里怎么用 |
|---|---|
| JWT | 登录后签发 token，三端请求统一鉴权 |
| RBAC | 后台接口用 ADMIN 角色限制 |
| Redis 秒杀库存 | 任务接单时用 Lua 原子预扣名额 |
| DB 原子扣减 | `WHERE slots_left > 0` 防超卖 |
| Redis 回补 | DB 失败时回补 Redis 名额 |
| 限流 | 验证码/登录接口用 Redis Lua 限流 |
| 缓存三防 | 任务列表缓存防穿透/击穿/雪崩 |
| 幂等 Token | 发布任务防重复提交 |
| 钱包原子更新 | `balance = balance + ?` 防丢失更新 |

## 30 秒介绍模板

面试或和别人介绍时，可以这样说：

> 蓝血菁英平台是一个面向 AI 技术人才的成长与任务撮合平台。我参与的是 Java 重构版本，项目包含前台 H5、后台管理、小程序和 Spring Boot 后端四端。核心业务是任务撮合闭环：企业发布任务，用户接单并按里程碑提交成果，雇主审核通过后按里程碑金额入账钱包。技术上用 Spring Boot、MyBatis-Plus、MySQL、Redis、JWT，并实现了 Redis Lua 秒杀预扣、DB 原子兜底、限流、缓存三防、幂等等能力。
