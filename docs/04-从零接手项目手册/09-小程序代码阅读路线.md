# 09 · 小程序代码阅读路线

> 目标：看懂 `apps/miniapp`，知道小程序在项目里负责什么。

---

## 一、小程序是什么？

目录：

```text
apps/miniapp
```

技术栈：

```text
uni-app + Vue3 + TypeScript + Pinia
```

定位：

```text
微信小程序端，是移动端轻量入口，不覆盖所有 H5 功能，重点覆盖任务、钱包、通知、我的。
```

---

## 二、小程序和 H5 的区别

| 维度 | 前台 H5 | 小程序 |
|---|---|---|
| 技术 | Vue3 + Vite + Vant | uni-app + Vue3 |
| 场景 | 完整用户端 | 移动轻量入口 |
| 功能 | 全量功能更多 | 任务/钱包/通知为主 |
| 登录 | 账号密码/邮箱/手机 | wx-login 桩/微信登录 |
| 运行 | 浏览器 | 微信开发者工具 |

---

## 三、建议阅读顺序

```text
1. package.json             看脚本
2. src/App.vue              应用入口
3. src/pages.json           小程序页面和 tabBar 配置
4. src/pages/               页面
5. src/stores/              状态
6. src/utils/               请求封装
```

## 六、小程序登录

项目里小程序支持：

```text
wx.login → /auth/wx-login
```

如果没有配置真实 AppID/Secret，后端会走演示桩，返回演示用户 token。

你要理解：

```text
小程序不是完全独立后端，它仍然调用同一套 Spring Boot API。
```

---

## 七、小程序请求链路

一般流程：

```text
uni 页面
→ 调用 request 工具
→ 自动带 token
→ 请求 /api/*
→ 后端返回 Result<T>
→ 前端解包 data
```

这和 H5 的 Axios 逻辑类似，只是换成 `uni.request`。
