# 08 · 后台管理端代码阅读路线

> 目标：看懂 `apps/admin-web`，理解后台管理端如何组织和调用 `/admin/*` 接口。

---

## 一、后台管理端是什么？

目录：

```text
apps/admin-web
```

定位：

```text
平台运营和管理员使用的后台系统。
```

管理员在这里做：

```text
用户管理、认证审核、企业审核、社区管理、课程管理、任务管理、里程碑审核、收益结算、提现审核、系统配置、日志查看。
```

---

## 二、建议阅读顺序

```text
1. package.json          看脚本和依赖
2. src/main.ts           应用入口
3. src/router/           后台路由
4. src/layout/           后台整体布局
5. src/views/login/      登录页
6. src/views/dashboard/  工作台
7. src/views/*           按业务模块看页面
8. src/api/              接口封装
9. src/stores/           登录状态
```

## 六、后台和后端的关系

后台主要调用后端 `/admin/*` 接口。

后端侧通常有：

```text
modules/admin/controller/*
modules/admin/service/*
```

大部分后台接口都有：

```java
@PreAuthorize("hasRole('ADMIN')")
```

这意味着：

```text
只登录普通用户没用，必须是 ADMIN 角色。
```

---

## 七、后台 API 调用规律

一般规律：

```text
页面组件 → api 方法 → axios 请求 → /admin/* 后端接口 → admin service → mapper → DB
```

例如提现审核：

```text
WithdrawManage.vue
→ admin finance API
→ POST /admin/wallet/withdraw/{id}/review
→ AdminWalletService.reviewWithdraw
→ 修改 withdraw_record
→ APPROVED 或 REJECTED
→ REJECTED 时 addBalance 回补余额
```
