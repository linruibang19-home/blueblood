## 二、角色能力总览

### 普通用户

普通用户能做：

```text
[ ] 登录注册
[ ] 浏览社区/发现
[ ] 浏览课程
[ ] 浏览任务大厅
[ ] 接任务
[ ] 查看我的任务
[ ] 提交里程碑
[ ] 查看钱包和收益流水
[ ] 申请提现
[ ] 查看通知
[ ] 编辑个人资料
```

普通用户不能做：

```text
[ ] 后台管理
[ ] 审核用户
[ ] 审核企业
[ ] 审核提现
[ ] 系统配置
```

### 企业用户

企业用户能做：

```text
[ ] 普通用户全部能力
[ ] 发布任务
[ ] 发布岗位
[ ] 发布黑客松
[ ] 查看我发布的任务
[ ] 审核接单者提交的里程碑
```

企业用户的关键限制：

```text
必须通过企业认证审核后，user_type 才会从 personal 变为 enterprise。
```

### 管理员

管理员能做：

```text
[ ] 登录后台
[ ] 用户管理
[ ] 实名认证审核
[ ] 企业认证审核
[ ] 社区管理
[ ] 课程/作业管理
[ ] 任务管理
[ ] 接单/订单管理
[ ] 里程碑审核兜底
[ ] 收益流水管理
[ ] 提现审核
[ ] 通知管理
[ ] 系统配置
[ ] 权限/日志管理
```

## 五、推荐演示路径 C：企业发布任务

如果已有企业用户账号，可以直接登录企业用户。

如果没有，则走：

```text
1. 前台注册企业账号
2. 提交企业认证申请
3. 后台 admin 审核通过
4. 企业用户重新登录/刷新状态
5. 前台进入企业控制台/任务发布页
6. 发布任务，配置里程碑和金额
7. 任务进入任务大厅
```

这条路径能证明：

```text
企业认证 → user_type=enterprise → 发布权限 → 任务模板 → Redis 库存初始化
```

## 六、推荐演示路径 D：提现审核

普通用户侧：

```text
1. 登录普通用户
2. 进入钱包
3. 发起提现申请
```

管理员侧：

```text
1. 登录后台 admin
2. 进入财务管理 → 提现管理
3. 查看提现申请
4. 选择 APPROVED 或 REJECTED
```

重点理解：

```text
提现申请时先扣余额，相当于冻结；
如果管理员驳回，需要把余额加回去。
```

不要一开始就打开代码。先演示业务，再讲代码。

## 八、角色和权限容易混淆的点

| 问题 | 正确理解 |
|---|---|
| USER 和 personal 是一回事吗？ | 不是。USER 是角色，personal 是用户类型。 |
| enterprise 是角色吗？ | 不是，是用户类型，表示企业用户。 |
| ADMIN 是用户类型吗？ | 不是，是角色。 |
| 企业用户能进后台吗？ | 不一定。后台需要 ADMIN 角色。 |
| 管理员一定是企业用户吗？ | 不一定。管理员是后台角色，和 user_type 是两套概念。 |
