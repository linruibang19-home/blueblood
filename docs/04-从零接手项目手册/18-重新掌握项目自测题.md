# 18 · 重新掌握项目自测题

> 目标：用题目检验自己是否真的重新接手了项目。

---

## 使用方式

不要一边看答案一边答。

建议：

```text
1. 先合上文档
2. 自己口头回答
3. 回答不出来再回去看对应文档
4. 最后能连续讲出来，才算掌握
```

---

## L1：项目全貌

能答出来说明你知道项目是干什么的。

```text
1. 蓝血菁英平台一句话定位是什么？
2. 项目有哪四端？分别在哪个目录？
3. 普通用户、企业用户、管理员分别能做什么？
4. 项目核心业务闭环是什么？
5. “成长 + 变现”在项目里分别对应哪些模块？
6. 前台 H5 和后台管理端的区别是什么？
7. 小程序在项目里承担什么角色？
```

合格标准：能在 3 分钟内说清楚项目定位和四端职责。

---

## L2：能跑通项目

能答出来说明你能重新启动和演示项目。

```text
1. Docker 完整栈用哪个 compose 文件启动？
2. 启动前为什么要先打后端 jar？
3. 前台 H5 地址是什么？
4. 后台管理地址是什么？
5. 后端 API 地址是什么？
6. Knife4j 文档地址是什么？
7. 默认普通用户账号是什么？
8. 默认管理员账号是什么？
9. MySQL 和 Redis 暴露端口分别是什么？
10. 前台 Mock 模式和 API 模式怎么切换？
```

合格标准：能独立启动项目，并完成一次登录和任务浏览。

---

能答出来说明你知道代码入口在哪里。

```text
1. Spring Boot 启动类在哪里？
2. 统一响应 Result 在哪里？
3. 全局异常处理在哪里？
4. SecurityConfig 做了什么？
5. JwtAuthenticationFilter 做了什么？
6. 当前登录用户 ID 怎么获取？
7. 登录接口入口在哪里？
8. 接单接口入口在哪里？
9. 任务发布逻辑在哪里？
10. 里程碑审核逻辑在哪里？
11. 钱包入账逻辑在哪里？
12. 提现审核逻辑在哪里？
13. 前台任务大厅页面在哪里？
14. 后台提现审核页面在哪里？
15. 小程序任务详情页面在哪里？
```

合格标准：能在 IDE 中快速跳到对应文件。

---

## L4：能讲清核心链路

能答出来说明你理解核心业务和技术。

```text
1. 登录后 JWT 是怎么生成和使用的？
2. 后续请求如何通过 token 找到当前用户？
3. ADMIN 权限是怎么限制的？
4. 企业用户和 ADMIN 角色有什么区别？
5. 发布任务时创建了哪些数据？
6. 里程碑模板和里程碑实例怎么区分？
7. 用户接单时 Redis 和 DB 分别做了什么？
8. Redis Lua 为什么能防超卖？
9. DB 原子 SQL 为什么还需要？
10. @Transactional 为什么不回滚 Redis？
11. 失败时为什么要回补 Redis？
12. 里程碑提交后状态怎么变化？
13. 雇主审核通过后钱包怎么入账？
14. 钱包入账怎么防重复？
15. 提现为什么申请时先扣余额？
16. 提现驳回为什么要退余额？
```

合格标准：能连续讲 10 分钟核心链路，不需要看文档。

---

## L5：能继续开发

能答出来说明你不仅会看，还能继续改。

```text
1. 如果新增“任务评价”功能，需要新增哪些表？
2. 新增后端模块的目录结构应该怎么设计？
3. 新增接口时应该经过哪些层？
4. 如果评价只能由接单者在订单 settled 后提交，权限应该在哪里校验？
5. 如果评价要在任务详情展示，前台要改哪些页面？
6. 如果后台要管理评价，后台要改哪些页面？
7. 如果评价影响用户信誉分，钱包/用户模块是否需要联动？
8. 如果新接口需要登录，SecurityConfig 要不要改？
9. 如果新表需要软删除，要加什么字段？
10. 如果新功能涉及重复提交，应该怎么做幂等？
```

合格标准：能说出一个新功能从 SQL 到前端页面的完整改动路径。

---

## 项目掌握评分

| 等级 | 标准 | 状态 |
|---|---|---|
| L1 | 能说清项目是什么 | 入门 |
| L2 | 能跑通项目 | 可演示 |
| L3 | 能找到核心代码 | 可阅读 |
| L4 | 能讲清核心链路 | 可面试 |
| L5 | 能继续开发新功能 | 真掌握 |

你的目标至少是：

```text
L4：能讲清核心链路
```

如果要真正变成自己的能力，目标是：

```text
L5：能继续开发新功能
```

---

## 最后一题：请你自己讲一遍

不看任何文档，连续讲 8-10 分钟：

```text
这个项目是什么？
四端分别是什么？
核心业务闭环是什么？
登录认证怎么走？
任务发布和接单怎么走？
Redis 在里面做了什么？
里程碑结算怎么入账？
如果我要继续加功能应该怎么做？
```

如果能讲出来，你就重新接手成功了。
