# 10 · 数据库从零理解

> 目标：不要被 44+ 张表吓住，按业务域理解数据库。

---

## 一、先建立数据库整体观

数据库名：

```text
blueblood_v2
```

主要特点：

```text
MySQL 8
utf8mb4
InnoDB
金额 DECIMAL(10,2)
统一 created_at / updated_at / deleted_at
软删除用 deleted_at
```

不要一开始逐行读 `init.sql`，先按业务域分组理解。

## 二、核心业务域和表

### 1. 用户域

负责：登录用户、资料、技能、角色、认证。

常见表：

```text
user
user_profile
user_skill
user_verification
sys_role
sys_user_role
user_level_log
user_credit_log
```

关系：

```text
user 1 —— 1 user_profile
user 1 —— n user_skill
user 1 —— n user_verification
user n —— n sys_role（通过 sys_user_role）
```

你要理解：

```text
user 是登录主体；
user_profile 是扩展资料；
sys_role/sys_user_role 决定 ADMIN/USER；
user_type 决定 personal/enterprise。
```

---

### 2. 任务域

负责：任务发布、接单、里程碑、审核。

常见表：

```text
task
task_category
task_skill
task_order
task_milestone
milestone_submission
milestone_review
```

核心关系：

```text
task 1 —— n task_skill
task 1 —— n task_milestone（模板，order_id IS NULL）
task 1 —— n task_order
task_order 1 —— n task_milestone（实例，order_id 不为空）
task_milestone 1 —— 1 milestone_submission
task_milestone 1 —— 1 milestone_review
```

关键理解：

```text
task 是任务本体；
task_order 是用户接单后的订单；
task_milestone 既可以是任务模板，也可以是订单实例；
通过 order_id 是否为空区分。
```

---

### 3. 钱包域

负责：余额、流水、提现。

常见表：

```text
wallet_account
wallet_record
withdraw_record
```

关系：

```text
user 1 —— 1 wallet_account
user 1 —— n wallet_record
user 1 —— n withdraw_record
```

关键理解：

```text
wallet_account 存余额；
wallet_record 存每一笔收入/提现流水；
withdraw_record 存提现申请；
```

幂等关键：

```text
wallet_record 通过 biz_type + biz_id 防重复入账。
```

---

### 4. 课程域

负责：课程、章节、学习进度。

常见表：

```text
course
course_chapter
course_progress
assignment
assignment_submission
assignment_grade
```

关系：

```text
course 1 —— n course_chapter
user 1 —— n course_progress
assignment 1 —— n assignment_submission
assignment_submission 1 —— 1 assignment_grade
```

关键理解：

```text
课程进度是线性模型：completed_chapters 只增不减。
```

### 6. 企业与机会域

负责：企业认证、岗位、黑客松。

常见表：

```text

```

关键理解：

```text

```

### 7. 系统域

负责：通知、配置、字典、文件、日志。

常见表：

```text

sys_dict
sys_file
sys_operation_log
```

关键理解：

```text
notification 是通知内容；
notification_read 记录用户是否已读；
sys_file 记录上传文件；
sys_operation_log 预留操作日志。
```

## 三、最重要的 ER 图

          ├── 1:n task_skill
          ├── 1:n task_milestone（模板）

## 四、任务表和订单表的区别

这是最容易混淆的点。

| 表 | 含义 | 什么时候创建 |
|---|---|---|
| `task` | 任务大厅里的任务 | 企业发布任务时 |
| `task_order` | 用户接单后的订单 | 用户点击接单时 |
| `task_milestone` 模板 | 任务本身配置的里程碑 | 发布任务时 |
| `task_milestone` 实例 | 某个订单下的里程碑 | 用户接单时复制模板 |

简单理解：

```text
task 是“招聘公告”；
task_order 是“某个用户接了这份工作”；
task_milestone 模板是“工作计划”；
task_milestone 实例是“这个用户自己的交付节点”。
```

## 六、软删除怎么理解？

项目多数表有：

```text
deleted_at
```

当一条数据被删除时，不是真正 `DELETE`，而是：

```text
设置 deleted_at = 当前时间
```

查询时需要过滤：

```text
deleted_at IS NULL
```

为什么这样做？

```text
1. 保留历史数据
2. 防止误删
3. 方便审计
```
