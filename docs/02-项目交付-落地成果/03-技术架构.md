# 03 · 技术架构

> 技术全貌 + **每个架构机制对应到具体代码**(类.方法 + 可点击文件路径),面试讲技术时能直接定位代码。

## 整体架构

```
┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│  前台 H5    │  │  后台管理   │  │  小程序     │   (Vue3 / uni-app)
│ user-web    │  │ admin-web   │  │ miniapp     │
│ :3000       │  │ :3001       │  │             │
└──────┬──────┘  └──────┬──────┘  └──────┬──────┘
       │  Axios / uni.request (Bearer JWT)
       └────────────────┼────────────────┘
                        ▼
              ┌──────────────────┐
              │  Spring Boot API │  :8090  (统一后端)
              │  /api/**         │
              └────┬───────┬─────┘
                   │       │
            ┌──────▼──┐ ┌──▼──────┐
            │ MySQL 8 │ │ Redis 7 │
            │ :3307   │ │ :6380   │
            └─────────┘ └─────────┘
```

四端共用 **一套后端 API**,前端只做视图与交互(前后端分离 + 多端复用)。

## 请求处理链路(以 `POST /task/{id}/accept` 为例,每环节配代码)

```
HTTP 请求(Bearer JWT)
  ①  CORS 预检/跨域
  ②  JwtAuthenticationFilter   解析 token → 存 SecurityContext
  ③  SecurityFilterChain       路径鉴权(/auth/** 放行,其余需认证)
  ④  @PreAuthorize             方法级角色检查(后台接口)
  ⑤  RateLimitAspect           @RateLimit 限流(若方法标注)
  ⑥  Controller                @Valid 参数绑定
  ⑦  Service                   业务逻辑(@Transactional 事务)
  ⑧  Mapper + Redis            SQL(自动软删过滤)+ 缓存
  ⑨  Result<T> 统一响应 / 异常走 GlobalExceptionHandler
```

| 环节 | 代码 |
|------|------|
| ① CORS | `SecurityConfig.corsConfigurationSource()` @ [SecurityConfig.java](../../server/blueblood-api/src/main/java/com/blueblood/api/config/SecurityConfig.java) |
| ② JWT 过滤器 | `JwtAuthenticationFilter.doFilterInternal()` → `JwtUtil.parse/generate` @ [JwtAuthenticationFilter.java](../../server/blueblood-api/src/main/java/com/blueblood/api/security/JwtAuthenticationFilter.java) · [JwtUtil.java](../../server/blueblood-api/src/main/java/com/blueblood/api/security/JwtUtil.java) |
| ③ 路径鉴权 | `SecurityConfig.securityFilterChain()`(`/auth/**`、`/health`、`/uploads` permitAll,其余 authenticated)@ [SecurityConfig.java](../../server/blueblood-api/src/main/java/com/blueblood/api/config/SecurityConfig.java) |
| ④ 方法鉴权 | `@PreAuthorize("hasRole('ADMIN')")`(admin 接口)· 业务级权限在 service 内(`TaskService.orderDetail` 判 owner/employer/admin) |
| ⑤ 限流 | `RateLimitAspect.around()` + Lua 原子计数 @ [RateLimitAspect.java](../../server/blueblood-api/src/main/java/com/blueblood/api/common/ratelimit/RateLimitAspect.java) |
| ⑥ Controller | `TaskController.accept()` @ [TaskController.java](../../server/blueblood-api/src/main/java/com/blueblood/api/modules/task/controller/TaskController.java) |
| ⑦ Service | `TaskService.accept()`(`@Transactional`)@ [TaskService.java](../../server/blueblood-api/src/main/java/com/blueblood/api/modules/task/service/TaskService.java) |
| ⑧ Mapper/Redis | `TaskMapper.decrementSlotsLeft` + `TaskStockService.tryDeduct`(Redis Lua)|
| ⑨ 响应/异常 | `Result.success()` @ [Result.java](../../server/blueblood-api/src/main/java/com/blueblood/api/common/result/Result.java) · `GlobalExceptionHandler` @ [GlobalExceptionHandler.java](../../server/blueblood-api/src/main/java/com/blueblood/api/common/exception/GlobalExceptionHandler.java) |

## 并发架构(本项目最大技术亮点,每机制配代码)

| 机制 | 实现 | 代码 |
|------|------|------|
| **Redis Lua 秒杀预扣** | Lua `GET判>0→DECR` 原子 + 懒加载 + 回补 | `TaskStockService.tryDeduct/rollback` @ [TaskStockService.java](../../server/blueblood-api/src/main/java/com/blueblood/api/modules/task/service/TaskStockService.java) |
| **DB 原子扣兜底** | `WHERE slots_left>0` 行锁防超卖 | `TaskMapper.decrementSlotsLeft` @ [TaskMapper.java](../../server/blueblood-api/src/main/java/com/blueblood-api/modules/task/mapper/TaskMapper.java) |
| **Redis/DB 双写一致性** | 预扣后 try-catch,失败回补 Redis | `TaskService.accept()` 的 try-catch @ [TaskService.java](../../server/blueblood-api/src/main/java/com/blueblood/api/modules/task/service/TaskService.java) |
| **Redis+Lua 限流** | `INCR+EXPIRE` 原子,`@RateLimit` 注解 + AOP | `RateLimitAspect` @ [RateLimitAspect.java](../../server/blueblood-api/src/main/java/com/blueblood/api/common/ratelimit/RateLimitAspect.java) · [RateLimit.java](../../server/blueblood-api/src/main/java/com/blueblood/api/common/ratelimit/RateLimit.java) |
| **缓存防击穿/穿透/雪崩** | 互斥锁 + 空值标记 + 随机TTL | `TaskListCacheService` @ [TaskListCacheService.java](../../server/blueblood-api/src/main/java/com/blueblood/api/modules/task/service/TaskListCacheService.java) |
| **幂等 Token** | Redis `DEL` 原子消费(1=首次/0=重复) | `IdempotentService.consume` @ [IdempotentService.java](../../server/blueblood-api/src/main/java/com/blueblood/api/common/idempotent/IdempotentService.java) |
| **原子更新防丢失更新** | `balance=balance+?` / `view_count=view_count+1` | `WalletAccountMapper.addBalance/deductBalance` @ [WalletAccountMapper.java](../../server/blueblood-api/src/main/java/com/blueblood/api/modules/wallet/mapper/WalletAccountMapper.java) · `TaskMapper.incrementViewCount` @ [TaskMapper.java](../../server/blueblood-api/src/main/java/com/blueblood-api/modules/task/mapper/TaskMapper.java) |

## 安全设计(每项配代码)

| 机制 | 说明 | 代码 |
|------|------|------|
| JWT 无状态认证 | token 携带 userId/username/role,Filter 解析 | `JwtAuthenticationFilter` @ [JwtAuthenticationFilter.java](../../server/blueblood-api/src/main/java/com/blueblood/api/security/JwtAuthenticationFilter.java) |
| RBAC 角色鉴权 | `@PreAuthorize("hasRole('ADMIN')")` + 角色表 | `sys_role`/`sys_user_role`,各 admin Controller |
| 业务级权限 | service 内判数据归属/user_type | `TaskService.orderDetail`(owner/employer/admin)、`EnterpriseService.ensureEnterprise`(user_type) |
| 401/403 JSON 化 | 不暴露堆栈 | `RestAuthenticationEntryPoint`/`RestAccessDeniedHandler` @ [security/](../../server/blueblood-api/src/main/java/com/blueblood/api/security/) |
| 密码加密 | BCrypt + `@JsonIgnore` | `SecurityConfig.passwordEncoder()` @ [SecurityConfig.java](../../server/blueblood-api/src/main/java/com/blueblood/api/config/SecurityConfig.java) |

## 数据流

- **MySQL**:全部业务主数据,44+ 表,统一软删(`deleted_at`,手动 `.isNull` 过滤)
- **Redis**(多用途):
  - 验证码:`verify:{type}:{target}` TTL 300s
  - 秒杀名额库存:`task:slots:{taskId}`(`TaskStockService`)
  - 任务列表缓存:`task:list:*`(`TaskListCacheService`)
  - 限流计数:`rate:*`(`RateLimitAspect`)
  - 幂等 token:`idem:*`(`IdempotentService`)

## 部署架构(Docker Compose)

```yaml
services:
  mysql:   :3307→3306   挂载 init.sql 自动建库
  redis:   :6380→6379
  api:     :8090→8080   从本地 jar 构建(COPY target/*.jar)
  user-web::3000→80     nginx 托管 dist
  admin-web::3001→80    nginx 托管 dist
```

一键 `docker compose -f deploy/docker-compose.full.yml up -d --build` 起全栈。详见 [05-部署运行手册](05-部署运行手册.md)。
