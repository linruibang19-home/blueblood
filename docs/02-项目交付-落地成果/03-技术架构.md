# 03 · 技术架构

## 整体架构

```
┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│  前台 H5    │  │  后台管理   │  │  小程序     │   (Vue3 / uni-app)
│ user-web    │  │ admin-web   │  │ miniapp     │
│ :3000       │  │ :3001       │  │             │
└──────┬──────┘  └──────┬──────┘  └──────┬──────┘
       │  Axios / uni.request (Bearer JWT)
       └────────────────┼────────────────┘
                        ▼
              ┌──────────────────┐
              │  Spring Boot API │  :8090  (统一后端)
              │  /api/**         │
              └────┬───────┬─────┘
                   │       │
            ┌──────▼──┐ ┌──▼──────┐
            │ MySQL 8 │ │ Redis 7 │
            │ :3307   │ │ :6380   │
            └─────────┘ └─────────┘
```

四端共用 **一套后端 API**,前端只做视图与交互。这是"前后端分离 + 多端复用"的标准架构。

## 后端分层(`com.blueblood.api`)

```
com.blueblood.api
├── common/          通用基建
│   ├── result/      Result<T> / ResultCode / PageResult / PageQuery
│   ├── exception/   BusinessException + GlobalExceptionHandler(统一异常→JSON)
│   └── entity/      BaseEntity(id/createdAt/updatedAt/deletedAt)
├── security/        JWT 全家桶(JwtUtil/Filter/EntryPoint/DeniedHandler/SecurityUtils)
├── config/          SecurityConfig / MybatisPlusConfig / RedisConfig / Knife4j / CORS / WebMvc
└── modules/         业务模块(每个独立包)
    ├── auth/        认证(登录/注册/验证码/wx-login)
    ├── user/        用户(资料/技能/认证/角色/等级信誉)
    ├── community/   社区(小组/帖子/评论/点赞/活动/聊天)
    ├── course/      课程(课程/章节/进度 - 线性模型)
    ├── assignment/  作业(提交/评分,答案审后才揭晓)
    ├── task/        任务(任务/分类/订单/里程碑/提交/审核)
    ├── wallet/      钱包(账户/流水/提现 + credit 入账)
    ├── notification/通知
    ├── enterprise/  企业(认证申请/岗位/黑客松发布)
    ├── system/      系统(文件/字典/配置/操作日志)
    └── admin/       后台(各模块的管理 Controller/Service,均 @PreAuthorize ADMIN)
```

每个业务模块内部统一 `controller / service / mapper / entity / dto(VO/Request)` 五层。

## 请求处理链路(一次 `GET /task/orders/{id}`)

```
HTTP 请求
  → CorsFilter                 跨域
  → JwtAuthenticationFilter    解析 Authorization: Bearer <token>
      → JwtUtil.parse          验签/过期检查
      → 构建 UsernamePasswordAuthenticationToken
      → 存入 SecurityContextHolder
  → SecurityFilterChain        路径鉴权(/auth/** 放行,其余需认证)
  → @PreAuthorize              方法级角色检查(后台接口)
  → Controller                 参数绑定 + @Valid
  → Service                    业务逻辑(校验/事务/调用 Mapper)
  → Mapper(MyBatis-Plus)       SQL,自动 .isNull(deletedAt) 软删过滤
  → MySQL / Redis
  → 返回 Result<T>             统一 {code,message,data,success}
  → 异常走 GlobalExceptionHandler 统一转 JSON
```

## 四端协作

| 能力 | user-web | admin-web | miniapp | api |
|------|----------|-----------|---------|-----|
| 注册登录 | ✅ 3 种方式 | ✅ 账密 | ✅ wx-login 桩 | 统一 `/auth/*` |
| 任务发布 | ✅ | ✅(后台代发) | — | `POST /task` |
| 接单/提交 | ✅ | — | ✅ | `/task/{id}/accept` |
| 雇主验收 | ✅ 雇主工作台 | ✅ 兜底审核 | — | `/task/milestones/{id}/review` |
| 钱包 | ✅ 查看/提现 | ✅ 结算/提现审核 | ✅ 查看 | `/wallet/*` |

## 安全设计

1. **JWT 无状态认证**:不存 session,token 携带 userId/username/role,签名防篡改。Filter 每次请求解析。
2. **RBAC**:角色表 + 方法级 `@PreAuthorize`,后台接口强制 ADMIN。
3. **业务级权限**:service 内校验所有权(订单只能 owner/雇主/admin 看)、user_type(发布限企业)。
4. **CORS**:配置允许跨域 + 携带凭证。
5. **入口/拒绝处理**:`RestAuthenticationEntryPoint`(401 JSON)+ `RestAccessDeniedHandler`(403 JSON),不暴露堆栈。
6. **敏感字段**:`User.password` 用 `@JsonIgnore` + BCrypt 存储。

## 数据流

- **MySQL**:全部业务主数据,44+ 表,统一软删(`deleted_at`,手动 `.isNull` 过滤)
- **Redis**:验证码(`verify:{type}:{target}`,TTL 300s);后续可扩展 token 黑名单/缓存

## 工程规范(全项目统一)

| 规范 | 实现 |
|------|------|
| 统一响应 | `Result.success(data)` / `Result.fail(code,msg)`,前端拦截器按 code 判断 |
| 统一异常 | `BusinessException(ResultCode, msg)` + `@RestControllerAdvice` 兜底 |
| 软删除 | `BaseEntity.deletedAt` + 查询手动 `.isNull`(**未启用 MyBatis-Plus 全局逻辑删除**,规避 datetime 字段坑,详见 [08-面试](08-面试问答准备.md)) |
| 分页 | `PageQuery` + `PageResult.of(IPage)`,统一 `{list,total,page,pageSize}` |
| 自动填充 | `MyMetaObjectHandler` 自动填 createdAt/updatedAt |
| 时间序列化 | `@JsonFormat` 统一 `yyyy-MM-dd HH:mm:ss` |
| 参数校验 | Bean Validation(`@NotBlank/@Min/@Pattern`)+ 全局异常转 400 |

## 部署架构(Docker Compose)

```yaml
services:
  mysql:   :3307→3306   挂载 init.sql 自动建库
  redis:   :6380→6379
  api:     :8090→8080   从本地 jar 构建(COPY target/*.jar)
  user-web::3000→80     nginx 托管 dist
  admin-web::3001→80    nginx 托管 dist
```

一键 `docker compose -f deploy/docker-compose.full.yml up -d --build` 起全栈。
