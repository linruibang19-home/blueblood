# 05 · 部署运行手册

## 前置

- **Docker Desktop**(Windows/Mac)或 Docker Engine + Docker Compose
- JDK 17 + Maven 3.9(仅重建后端 jar 时需要)
- Node 18+(仅重建前端时需要)

## 一键启动(Docker 全栈)

在项目根目录:

```bash
# 首次或后端代码变更:重新打 jar(后端 Dockerfile 用 COPY target/*.jar)
cd server/blueblood-api
mvn -q clean package -DskipTests
cd ../..

# 启动全部 5 个服务(mysql 会自动导入 init.sql 建库)
docker compose -f deploy/docker-compose.full.yml up -d --build
```

## 服务端口

| 服务 | 容器 | 宿主端口 | 访问 |
|------|------|---------|------|
| 后端 API | blueblood-v2-api | **8090** | http://localhost:8090/api |
| 接口文档 | (api) | 8090 | http://localhost:8090/api/doc.html |
| 前台 H5 | blueblood-v2-user-web | **3000** | http://localhost:3000 |
| 后台管理 | blueblood-v2-admin-web | **3001** | http://localhost:3001 |
| MySQL | blueblood-v2-mysql | 3307 | root / blueblood |
| Redis | blueblood-v2-redis | 6380 | — |

> 端口均错开本机原生服务(mysql→3307, redis→6380, api→8090),避免冲突。

## 重建单个服务

```bash
# 仅后端(改了 Java 后)
cd server/blueblood-api && mvn -q clean package -DskipTests
docker compose -f deploy/docker-compose.full.yml up -d --build api

# 仅前端
docker compose -f deploy/docker-compose.full.yml up -d --build user-web   # 或 admin-web
```

## 常见问题

| 问题 | 解决 |
|------|------|
| 8080 端口被占 | api 用 `SERVER_PORT` 环境变量走 8090,容器内仍是 8080 |
| 改了后端 Java 不生效 | Dockerfile 是 `COPY target/*.jar`,必须先 `mvn package` 再 build api |
| 改了前端不生效 | 前端是 nginx 托管编译产物,需 `--build user-web` 重建镜像 |
| MySQL 启动首次慢 | init.sql 导入 44 表 + 数据,等 healthcheck 通过(api depends_on healthy) |
| 中文乱码 | 库/表统一 utf8mb4;Windows curl POST 中文用 ASCII 转义或文件 |
| 成长页空白 | 课程接口需登录,未登录 401 → 先登录 |

## 停止 / 清理

```bash
docker compose -f deploy/docker-compose.full.yml down       # 停服务(保留数据卷)
docker compose -f deploy/docker-compose.full.yml down -v    # 连同 mysql 数据卷一起删(慎)
```
