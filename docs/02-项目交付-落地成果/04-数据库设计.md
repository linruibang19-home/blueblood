# 04 · 数据库设计

数据库 `blueblood_v2`(与原 Laravel 版 `blueblood` 库共存,互不干扰)。

## 表分组(44+ 表,按业务域)

| 业务域 | 表 | 说明 |
|--------|-----|------|
| **账户** | `user` / `user_profile` / `user_skill` / `sys_role` / `sys_user_role` / `user_verification` / `user_level_log` / `user_credit_log` | 用户主表 + 档案 + 技能 + RBAC + 实名认证 + 等级/信誉日志 |
| **社区** | `interest_group` / `post` / `post_comment` / `post_like` / `post_favorite` / `activity` / `activity_signup` / `chat_session` / `chat_message` | 小组/帖子/互动/活动/聊天 |
| **教育** | `course` / `course_chapter` / `course_progress` / `assignment` / `assignment_submission` / `assignment_grade` | 课程/章节/进度/作业/提交/评分 |
| **任务** | `task` / `task_category` / `task_skill` / `task_order` / `task_milestone` / `milestone_submission` / `milestone_review` | 任务/分类/技能/订单/里程碑/提交/审核 |
| **财务** | `wallet_account` / `wallet_record` / `withdraw_record` | 账户/流水/提现 |
| **企业** | `enterprise_application` / `job` / `hackathon` | 认证申请/岗位/黑客松 |
| **系统** | `sys_file` / `sys_dict` / `sys_config` / `sys_operation_log` / `notification` / `notification_read` | 文件/字典/配置/操作日志/通知 |

## 全局约定

- **主键**:`id BIGINT UNSIGNED AUTO_INCREMENT`
- **审计字段**(全部表继承 `BaseEntity`):`created_at` / `updated_at`(自动维护)/ `deleted_at`(软删)
- **软删除**:`deleted_at DATETIME NULL`,**查询手动 `.isNull(Entity::getDeletedAt)`** —— 未启用 MyBatis-Plus 全局逻辑删除(规避 datetime 字段坑,见 [08-面试](08-面试问答准备.md))
- **字符集**:`utf8mb4 / utf8mb4_unicode_ci`
- **时间序列化**:`@JsonFormat(pattern="yyyy-MM-dd HH:mm:ss")`

## 核心 ER(任务结算闭环)

```
user (employer_id) ──1:N──▶ task ──1:N──▶ task_order ──1:N──▶ task_milestone
   │                          │              │ userId(接单者)        │
   │                       task_skill        │                  ┌────┴────┐
   │                                       wallet_account   milestone   milestone
   │                                          │            _submission  _review
   └──1:1──▶ wallet_account ──1:N──▶ wallet_record (type=income, biz_type=task, biz_id=milestone_id)
```

**任务结算的关键表关系**:
- `task.employer_id` 记录发布者(雇主),验收时校验 `task.employer_id == 当前用户`
- `task_milestone` 用 `order_id IS NULL` 表示**任务级模板**,接单时复制为实例(`order_id` 填订单 id)
- `task_milestone.reward` 是每里程碑独立金额,审核 APPROVED 即入账
- `wallet_record(biz_type='task', biz_id=milestone_id)` 做结算幂等去重

## 关键唯一约束(并发安全基石)

| 表 | 唯一键 | 作用 |
|----|--------|------|
| `task_order` | `uk_order_pair(task_id, user_id)` | 同一用户不能重复接同一任务 |
| `wallet_account` | `uk_wallet_user(user_id)` | 一人一账户 |
| `milestone_submission` | `uk_ms_milestone(milestone_id)` | 一里程碑一提交(upsert) |
| `task_category` | `uk_tc_name(name)` | 分类名唯一 |
