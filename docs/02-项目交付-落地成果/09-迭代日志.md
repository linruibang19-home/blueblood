## 经验沉淀

- **先验证再扩展**:每期先跑通端到端,再加功能
- **bug 即资产**:每个真实 bug 的定位过程都是面试素材(资损/超卖/401)
- **不过度设计**:Redis 只在必须时用、并发用 DB 兜底而非分布式锁、软删除手动而非全局
- **工程规范优先**:统一响应/异常/分页让后续开发线性提速
- **文档随项目演进**:开发方案(前)与落地成果(后)分离,既可追溯又能直接用于简历
