# 02 · 业务架构

## 平台定位

连接 **AI 人才**(学习者/接单者)与 **企业/平台**(任务发布者/雇主),通过"成长 + 变现"双轮驱动留存用户。

## 业务域划分

平台按 **6 个业务域** 组织,前台 4 个 tab 与后台 6 个分组通过业务域一一映射:

| 业务域 | 前台 | 后台分组 | 说明 |
|--------|------|---------|------|
| **社区** | 发现 | 社区管理 | 小组/帖子/精英/聊天 |
| **教育** | 成长 | 教育管理 | 课程/作业(纯学习) |
| **机会** | 任务 | 机会管理 | 任务 + 黑客松 + 岗位(变现机会聚合) |
| **财务** | 我的·钱包 | 财务管理 | 余额/流水/提现 |
| **账户** | 我的·资料 | 平台运营·用户 | 资料/认证/设置 |
| **运营**(后台专属) | — | 工作台 + 平台运营 | 总览/用户/权限/配置/日志/通知/企业审核 |

> 设计原则:**前台按用户场景聚合,后台按业务域分组,用同一套域名词做映射桥梁。** 不追求 1:1,但每个功能都有唯一归属。

## 用户角色与权限

| 角色 | 标识 | 能做什么 |
|------|------|---------|
| **个人用户** | `user_type=personal` | 学习、接单、社区、提现 |
| **企业用户** | `user_type=enterprise`(认证通过后) | 个人全部能力 + 发布任务/岗位/黑客松 + 雇主验收 |
| **管理员** | `role=ADMIN` | 后台全部管理 + 兜底审核 |

权限两层:
- **RBAC**(角色):`sys_role` + `sys_user_role`,后台接口 `@PreAuthorize("hasRole('ADMIN')")`
- **user_type**(用户类型):发布类接口在 service 内校验 `user_type=enterprise`,不用注解(因不限角色而限类型)

## 核心业务流程

### 流程一:任务撮合 + 分阶段结算(平台核心闭环)

```
企业/个人发布任务(配多里程碑,各带金额,reward=Σ里程碑金额)
        │
        ▼
任务进大厅(APPROVED) ── 用户接单 ──▶ 生成订单 + 复制里程碑实例
        │                                   │
        │                                   ▼
        │                            接单者提交里程碑(成果+附件)
        │                                   │
        ▼                                   ▼
   雇主(发布者)审核 ◀─────────────────────┘
        │
   ┌────┴────┐
   APPROVED  REJECTED → 里程碑不计入进度,接单者可重提
   │
   ▼
按里程碑 reward 入账接单者钱包(income,幂等)
   │
   ▼
全部里程碑 APPROVED → 订单 settled(结算完成)
```

**关键设计点**(详见 [07-亮点](07-项目亮点与简历素材.md)、[08-面试](08-面试问答准备.md)):
- 接单原子 SQL 防超卖 + 唯一索引防重复接单
- 分阶段结算幂等(钱包流水按 bizType+bizId 去重)
- REJECTED 不计入进度分母(避免订单卡死)

### 流程二:企业认证

```
企业注册(提交营业执照/信用代码) → 企业申请 PENDING
        → admin 审核 APPROVED → user_type 置 enterprise
        → 可发布岗位/黑客松/任务
```

### 流程三:课程学习(线性进度)

```
选课 → 章节列表(按 chapter_order) → 学习第 k 章 → 标记完成
     → completed_chapters = max(原值, k) → 重算 progress% / status
```

### 流程四:钱包提现

```
用户申请提现 → 余额冻结(balance-=amount) → 提现记录 PENDING
     → admin 审核:APPROVED → withdrawn / REJECTED → 余额退回(修复了"驳回不退款"的资损 bug)
```

## 前后台模块对应关系(清晰映射)

```
前台 4 tab              后台 6 组
─────────────────────────────────────
发现(社区)      ↔  社区管理(小组/帖子)
成长(课程)      ↔  教育管理(课程/作业)
任务(机会)      ↔  机会管理(任务/接单/里程碑/黑客松/岗位)
我的(钱包)      ↔  财务管理(结算/提现)
                 ↘  工作台 + 平台运营(后台专属基础设施)
```

**对应关系一览**:

| 前台功能 | 后台管理 |
|---------|---------|
| 发现·小组/帖子 | 小组管理 / 帖子管理 |
| 成长·课程 | 课程管理 / 作业管理 |
| 任务·任务大厅 | 任务管理(发布/下架) |
| 任务·我的任务 | 接单管理(查看/强制关闭) |
| 任务·雇主审核 | 里程碑审核(后台兜底) |
| 任务·黑客松/岗位 | 黑客松管理 / 岗位管理 |
| 我的·钱包 | 收益结算 / 提现管理 |
| 我的·企业认证 | 企业认证审核 |
| 我的·资料 | 用户管理 / 认证审核 |
| — | 通知管理 / 权限管理 / 系统配置 / 操作日志 |
