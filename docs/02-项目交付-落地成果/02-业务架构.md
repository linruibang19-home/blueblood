# 02 · 业务架构

> 平台业务全貌 + **每个核心流程逐步拆解并对应到具体代码**(类.方法 + 可点击文件路径),面试讲业务时能直接定位代码。

## 平台定位

连接 **AI 人才**(学习者/接单者)与 **企业/平台**(任务发布者/雇主),通过"成长 + 变现"双轮驱动留存用户。

## 业务域划分(6 域,前台 4 tab ↔ 后台 6 组)

| 业务域 | 前台 | 后台分组 | 说明 |
|--------|------|---------|------|
| **社区** | 发现 | 社区管理 | 小组/帖子/精英/聊天 |
| **教育** | 成长 | 教育管理 | 课程/作业(纯学习) |
| **机会** | 任务 | 机会管理 | 任务 + 黑客松 + 岗位(变现机会聚合) |
| **财务** | 我的·钱包 | 财务管理 | 余额/流水/提现 |
| **账户** | 我的·资料 | 平台运营·用户 | 资料/认证/设置 |
| **运营**(后台专属) | — | 工作台 + 平台运营 | 总览/用户/权限/配置/日志/通知/企业审核 |

> 设计原则:**前台按用户场景聚合,后台按业务域分组,用同一套域名词做映射桥梁。** 每个功能唯一归属。

## 用户角色与权限

| 角色 | 标识 | 能做什么 |
|------|------|---------|
| **个人用户** | `user_type=personal` | 学习、接单、社区、提现 |
| **企业用户** | `user_type=enterprise`(认证后) | 个人全部 + 发布任务/岗位/黑客松 + 雇主验收 |
| **管理员** | `role=ADMIN` | 后台全部管理 + 兜底审核 |

权限两层:
- **RBAC**(角色):`sys_role` + `sys_user_role`,后台接口 `@PreAuthorize("hasRole('ADMIN')")`
- **user_type**(类型):发布类接口 service 内校验 `user_type=enterprise`(按类型非角色,不用注解)

## 核心业务流程(逐步 + 代码对应)

### 流程一:任务撮合 + 分阶段结算 ⭐ 平台核心闭环

> 6 个阶段,每阶段对应代码。这是面试讲业务的头号素材。

```
①发布任务 → ②接单(秒杀) → ③提交里程碑 → ④雇主审核 → ⑤分阶段入账 → ⑥订单 settled
```

**① 雇主发布任务(配多里程碑,各带金额)**
- 幂等校验(防重复提交):`IdempotentService.consume("publish", token)` @ [IdempotentService.java](../../server/blueblood-api/src/main/java/com/blueblood/api/common/idempotent/IdempotentService.java)
- 校验 `reward == Σ里程碑金额`(守恒)
- 建 `Task`(status=APPROVED 直接上架)+ `task_skill` + 里程碑模板(`task_milestone.order_id IS NULL`)
- 初始化 Redis 名额库存:`TaskStockService.initStock(taskId, slots)` @ [TaskStockService.java](../../server/blueblood-api/src/main/java/com/blueblood/api/modules/task/service/TaskStockService.java)
- 清任务大厅缓存:`TaskListCacheService.evictAll()`
- **入口**:`TaskPublishService.publish(req, token)` @ [TaskPublishService.java](../../server/blueblood-api/src/main/java/com/blueblood/api/modules/task/service/TaskPublishService.java)

**② 用户接单(Redis Lua 秒杀预扣 + DB 兜底)**
- Redis Lua 原子预扣名额:`TaskStockService.tryDeduct(taskId)`(Lua: GET判>0→DECR)
- 校验任务状态/重复接单(`hasOrder`)/用户等级
- 建 `TaskOrder`(`uk_order_pair` 兜底重复接单)+ 复制里程碑模板为实例
- DB 兜底原子扣名额:`TaskMapper.decrementSlotsLeft(taskId)`(`WHERE slots_left>0`)
- 失败(重复/满)→ `TaskStockService.rollback` 回补 Redis(双写一致性)
- **入口**:`TaskService.accept(taskId)` @ [TaskService.java](../../server/blueblood-api/src/main/java/com/blueblood/api/modules/task/service/TaskService.java)

**③ 接单者提交里程碑(成果 + 附件)**
- 状态校验(非 SUBMITTED/APPROVED 才可提交)
- upsert `MilestoneSubmission`(githubUrl/description/attachments)
- 里程碑 → `SUBMITTED`;重置审核为 PENDING
- **入口**:`MilestoneService.submit(milestoneId, req)` @ [MilestoneService.java](../../server/blueblood-api/src/main/java/com/blueblood/api/modules/task/service/MilestoneService.java)

**④ 雇主审核里程碑(雇主自助验收 + 分阶段结算触发点)**
- 权限:经 milestone→order→task 链路取 `task.employerId`,放行雇主或 ADMIN
- 状态白名单(`APPROVED`/`REJECTED`)
- 写 `MilestoneReview` + 里程碑状态流转
- **APPROVED → 触发⑤入账**;`recomputeOrderProgress` 判定⑥
- **入口**:`MilestoneService.review(milestoneId, req)` @ [MilestoneService.java](../../server/blueblood-api/src/main/java/com/blueblood/api/modules/task/service/MilestoneService.java)

**⑤ 分阶段结算入账(幂等)**
- 幂等:查 `wallet_record(biz_type='task', biz_id=里程碑id, type='income')` 存在则跳过
- 原子加余额:`WalletAccountMapper.addBalance(userId, amount)`(`balance=balance+?`,行锁防丢失更新)
- 写 `wallet_record`(type=income, status=available)
- **入口**:`WalletService.credit(userId, amount, "task", bizId, remark)` @ [WalletService.java](../../server/blueblood-api/src/main/java/com/blueblood/api/modules/wallet/service/WalletService.java)

**⑥ 订单结算完成**
- `recomputeOrderProgress`:有效里程碑(排除 REJECTED)全部 APPROVED → 订单 `settled`
- REJECTED 不计入进度分母(避免订单卡死 <100%)
- **代码**:`MilestoneService.recomputeOrderProgress(orderId)` @ [MilestoneService.java](../../server/blueblood-api/src/main/java/com/blueblood/api/modules/task/service/MilestoneService.java)

### 流程二:注册登录(用户名 / 邮箱+验证码 / 手机+验证码)

```
①发送验证码(Redis,限流) → ②注册(校验验证码) → ③登录(JWT) → ④/me 鉴权
```

- **① 发验证码**:`VerificationCodeService.sendCode(target, type)` @ [VerificationCodeService.java](../../server/blueblood-api/src/main/java/com/blueblood/api/modules/auth/service/VerificationCodeService.java)
  - Redis 存 `verify:{type}:{target}` TTL 300s;`@RateLimit(60s/1次)` 限流防刷
- **② 注册**:`AuthService.register(req)` @ [AuthService.java](../../server/blueblood-api/src/main/java/com/blueblood-api/modules/auth/service/AuthService.java)
  - email/phone 注册校验验证码(`VerificationCodeService.verify`);username 注册直通
  - `UserService.register` 建 User + Profile + 分配 USER 角色
- **③ 登录**:`AuthService.login(req)` → BCrypt 校验 → `JwtUtil.generate` 签发 token @ [AuthService.java](../../server/blueblood-api/src/main/java/com/blueblood/api/modules/auth/service/AuthService.java)
- **④ 当前用户**:`SecurityUtils.currentLoginUser()`(@PreAuthorize 后从 SecurityContext 取)

### 流程三:企业认证(注册后解锁发布能力)

```
①企业注册(材料) → ②提交认证申请(PENDING) → ③admin 审核 → ④APPROVED 置 enterprise → ⑤可发布任务/岗位/黑客松
```

- **①② 提交申请**:`EnterpriseService.submitApplication(req)` @ [EnterpriseService.java](../../server/blueblood-api/src/main/java/com/blueblood/api/modules/enterprise/service/EnterpriseService.java)
  - 校验已是 enterprise/已有 PENDING 申请则拒;建 `enterprise_application`(PENDING)
  - 前端流程:注册账号 → 自动登录拿 token → 提交申请([EnterpriseRegisterPage.vue](../../apps/user-web/src/pages/auth/EnterpriseRegisterPage.vue))
- **③④ 审核**:`AdminEnterpriseService.review(applicationId, req)` @ [AdminEnterpriseService.java](../../server/blueblood-api/src/main/java/com/blueblood/api/modules/admin/service/AdminEnterpriseService.java)
  - APPROVED → `UPDATE user SET user_type='enterprise'`(解锁发布)
- **⑤ 发布鉴权**:发布类接口 service 内校验 `user_type=enterprise`(`EnterpriseService.ensureEnterprise`)

### 流程四:课程学习(线性进度模型)

```
①选课 → ②学第 k 章 → ③标记完成 → ④completed_chapters=max(原,k) → ⑤重算 progress%/status
```

- **核心**:`CourseService.updateProgress(courseId, req)` @ [CourseService.java](../../server/blueblood-api/src/main/java/com/blueblood/api/modules/course/service/CourseService.java)
  - 线性模型:`completed_chapters = max(原值, chapter_order)`(只前进不回退)
  - `recompute`:progress% = round(completed/total×100);status = not_started/in_progress/completed
  - `currentChapter`:返回第一个未完成章节(== completedChapters+1)

---

### 流程五:钱包提现(含驳回退款资损修复)

```
①申请提现(余额冻结) → ②提现记录 PENDING → ③admin 审核 → APPROVED:withdrawn / REJECTED:余额退回
```

- **① 申请**:`WalletService.createWithdraw(req)` @ [WalletService.java](../../server/blueblood-api/src/main/java/com/blueblood/api/modules/wallet/service/WalletService.java)
  - 原子扣余额冻结:`WalletAccountMapper.deductBalance(userId, amount)`(`WHERE balance>=amount`,影响行数=0 则余额不足)
  - 建 `withdraw_record`(pending)+ `wallet_record`(type=withdraw, status=withdrawing)
- **③ 审核**:`AdminWalletService.reviewWithdraw(id, req)` @ [AdminWalletService.java](../../server/blueblood-api/src/main/java/com/blueblood/api/modules/admin/service/AdminWalletService.java)
  - **REJECTED → 余额退回**(`addBalance` 回补)—— 修复了原"驳回不退款"的资损 bug
  - APPROVED → withdrawn
