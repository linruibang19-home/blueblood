# 09-前台H5验收清单

## 任务目标

对第一阶段前台 H5 进行整体验收和收尾。

## 开发范围

允许修改：

```text
apps/user-web/
deploy/
README.md
```

## 前置依赖

必须完成：

```text
02-工程初始化任务
03-全局布局与基础组件任务
04-Mock数据与API封装任务
05-发现模块开发任务
06-成长模块开发任务
07-任务模块开发任务
08-我的模块开发任务
```

## 功能验收

必须通过：

```text
底部四个 Tab 可正常切换
发现模块完整可演示
成长模块完整可演示
任务模块完整可演示
我的模块完整可演示
所有子页面可返回
所有主要按钮有状态变化或 Toast
Mock 数据集中管理
API 封装层可替换真实接口
```

## 页面验收

必须检查：

```text
375px 移动端宽度
390px 移动端宽度
430px 最大容器宽度
深色主题一致
卡片间距一致
底部 Tab 不遮挡内容
顶部返回栏不遮挡内容
输入栏固定页面可正常滚动
```

## 工程验收

必须通过：

```text
npm install
npm run dev
npm run build
```

如配置了 lint：

```text
npm run lint
```

## Docker 验收

第一阶段需提供：

```text
apps/user-web/Dockerfile
deploy/docker-compose.local.yml
```

Docker 验收：

```text
docker compose -f deploy/docker-compose.local.yml up --build
浏览器可访问前台 H5
```

## 文档验收

补充或更新：

```text
apps/user-web/README.md
```

内容包括：

```text
本地开发命令
构建命令
Docker 运行命令
项目目录说明
Mock 数据说明
后续接后端说明
```

## 禁止事项

```text
不要在验收阶段重写业务模块
不要引入后端依赖
不要破坏已完成页面
```

## 交给 AI IDE 的执行提示

```text
请对第一阶段前台 H5 做整体验收和收尾。检查四大模块、所有子页面、Mock 数据、build、Docker 运行和 README。只修复验收发现的问题，不要新增后端或后台功能。
```
