## 一、JVM 内存结构

### 1.1 运行时数据区

```
┌─────────────────────────────────────────────────────────────────┐
│                         JVM 进程                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │                      堆（Heap）                           │  │
│  │  ┌────────────┐  ┌────────────┐                         │  │
│  │  │ Eden区     │  │ Survivor区 │                         │  │
│  │  │ (8/10)    │  │ S0   S1   │                         │  │
│  │  └────────────┘  └────────────┘                         │  │
│  │  ┌────────────────────────────────────────────┐         │  │
│  │  │                Old Gen（老年代）              │         │  │
│  │  └────────────────────────────────────────────┘         │  │
│  │                                                          │  │
│  │  存储：对象实例、数组                                      │  │
│  └──────────────────────────────────────────────────────────┘  │
│                                                                  │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │                     方法区（Method Area）                  │  │
│  │  存储：类信息、运行时常量池、静态变量、JIT 编译后代码       │  │
│  └──────────────────────────────────────────────────────────┘  │
│                                                                  │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │  虚拟机栈（VM Stack）    │  本地方法栈（Native Stack）   │  │
│  │  方法调用：局部变量表     │  本地方法调用                  │  │
│  │  操作数栈、返回地址       │                               │  │
│  └──────────────────────────────────────────────────────────┘  │
│                                                                  │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │                    程序计数器（PC Register）              │  │
│  │  当前线程执行的字节码行号                                 │  │
│  └──────────────────────────────────────────────────────────┘  │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 各区域作用

| 区域 | 存储内容 | 线程私有/共享 | 异常 |
|------|---------|-------------|------|
| **堆** | 对象实例、数组 | 共享 | OutOfMemoryError |
| **方法区** | 类信息、运行时常量池 | 共享 | OutOfMemoryError |
| **虚拟机栈** | 局部变量、操作数栈、返回地址 | 私有 | StackOverflowError |
| **本地方法栈** | Native 方法 | 私有 | StackOverflowError |
| **程序计数器** | 字节码行号 | 私有 | 无 |

### 1.3 堆内存划分

```
┌─────────────────────────────────────────────────────────────┐
│                        堆 Heap                               │
├────────────────────────────┬──────────────────────────────┤
│                            │                                │
│         Eden 区 (8/10)      │         Old Gen (2/10)       │
│  ┌──────────────────────┐  │  ┌──────────────────────────┐  │
│  │    Survivor S0       │  │  │                          │  │
│  │    (1/10)           │  │  │                          │  │
│  ├──────────────────────┤  │  │                          │  │
│  │    Survivor S1       │  │  │                          │  │
│  │    (1/10)           │  │  │                          │  │
│  └──────────────────────┘  │  │                          │  │
│                             │  │                          │  │
│  新对象分配在这里           │  │ 长期存活对象晋升在这里      │  │
│                             │  │                          │  │
└─────────────────────────────┴──┴──────────────────────────┘─┘
```

## 二、垃圾回收（GC）

### 2.1 判断对象可回收

| 方法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **引用计数法** | 有引用就+1，为0就回收 | 简单 | 无法处理循环引用 |
| **可达性分析** | 从 GC Roots 向下搜索，不可达就回收 | 能处理循环引用 | 需要停顿 |

**GC Roots 包含**：
- 虚拟机栈中引用的对象
- 方法区中静态属性引用的对象
- 方法区中常量引用的对象
- 本地方法栈中 JNI 引用的对象

### 2.2 垃圾回收算法

| 算法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **标记-清除** | 标记可回收 → 清除 | 简单 | 产生内存碎片 |
| **复制** | 分两半，用一半清一半 | 无碎片，高效 | 浪费一半空间 |
| **标记-整理** | 标记 → 整理到一端 → 清除 | 无碎片 | 移动成本高 |

**分代收集**：
```
Young Gen（年轻代）：
  ├─ Eden 区：新对象分配
  ├─ Survivor S0：第一次 Minor GC 后存活
  └─ Survivor S1：第二次 Minor GC 后存活
  
Old Gen（老年代）：长期存活对象
```

### 2.3 垃圾收集器

| 收集器 | 区域 | 算法 | 特点 | 适用 |
|--------|------|------|------|------|
| **Serial** | 单线程 | 复制 | 简单高效 | 单核 Client |
| **ParNew** | 多线程 | 复制 | 新生代并行 | 多核 Server |
| **Parallel Scavenge** | 多线程 | 复制 | 吞吐量优先 | 后台批处理 |
| **Serial Old** | 单线程 | 标记-整理 | — | 单核 Client |
| **Parallel Old** | 多线程 | 标记-整理 | 吞吐量优先 | 后台批处理 |
| **CMS** | 多线程 | 标记-清除 | 低停顿 | 互联网站 |
| **G1** | 多线程 | 标记-整理+复制 | 可预测停顿 | 所有场景 |

**CMS 收集过程**：
```
1. 初始标记（STW）：标记 GC Roots 直接引用的对象
2. 并发标记：遍历 GC Roots 图
3. 重新标记（STW）：修正并发标记变动
4. 并发清除：清除死亡对象
```

**G1 特点**：
- 把堆分成多个大小相等的 Region
- 可预测停顿（`-XX:MaxGCPauseMillis`）
- 优先回收价值最大的 Region

## 三、类加载

### 3.1 类加载过程

```
加载（Loading）→ 验证（Verification）→ 准备（Preparation）
     → 解析（Resolution）→ 初始化（Initialization）
```

| 阶段 | 说明 |
|------|------|
| **加载** | 通过类的全限定名获取字节码，在堆生成 Class 对象 |
| **验证** | 验证字节码格式、语义、符号引用 |
| **准备** | 为类变量分配内存，设置初始值（static） |
| **解析** | 符号引用 → 直接引用（类、方法、字段） |
| **初始化** | 执行 `<clinit>`，给类变量赋实际值 |

### 3.2 双亲委派模型

```
┌─────────────────────────┐
│    Bootstrap ClassLoader │  ← 加载 Java 核心类库（rt.jar）
└────────────┬────────────┘
             │ 委派
┌────────────▼────────────┐
│    Extension ClassLoader │  ← 加载扩展类库（jre/lib/ext）
└────────────┬────────────┘
             │ 委派
┌────────────▼────────────┐
│   Application ClassLoader │  ← 加载用户类路径（classpath）
└────────────┬────────────┘
             │ 委派
       ┌─────▼─────┐
       │ 自定义加载器 │
       └───────────┘
```

**为什么需要双亲委派？**
1. **安全性**：防止核心类被篡改（用户自定义 java.lang.String 不生效）
2. **类的唯一性**：保证类只加载一次
3. **避免重复加载**：父加载器加载过的，子加载器不再加载

### 3.3 打破双亲委派

| 场景 | 打破方式 |
|------|---------|
| JDBC 驱动 | SPI 机制（ServiceLoader） |
| Tomcat | 自定义 WebappClassLoader |
| 热部署 | 自定义 ClassLoader |

## 四、JVM 调优

### 4.1 常用参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `-Xms` | 堆初始大小 | `-Xms256m` |
| `-Xmx` | 堆最大大小 | `-Xmx512m` |
| `-Xss` | 栈大小 | `-Xss1m` |
| `-Xmn` | 年轻代大小 | `-Xmn128m` |
| `-XX:+UseG1GC` | 使用 G1 收集器 | — |
| `-XX:MaxGCPauseMillis` | 最大 GC 停顿时间 | `-XX:MaxGCPauseMillis=200` |

### 4.2 排查工具

| 工具 | 用途 |
|------|------|
| **jps** | 查看 Java 进程 |
| **jstat** | 查看 GC 统计 |
| **jmap** | 生成堆转储快照（heapdump） |
| **jstack** | 查看线程快照 |
| **jinfo** | 查看/修改 JVM 参数 |
| **MAT** | 分析 heapdump（内存分析） |
| **Arthas** | 在线诊断工具 |

### 4.3 常见问题

**OOM（OutOfMemoryError）**：
```
1. 堆溢出：对象太多无法回收
   → 增加堆大小、分析 heapdump

2. 栈溢出：递归调用太深
   → 减少栈大小（-Xss）或改递归为循环

3. 方法区溢出：类太多（ASM/CGLib）
   → 增加方法区大小
```

## 五、面试必问问题

### Q1：JVM 内存结构是怎样的？

**标准回答**：
> JVM 运行时数据区包括：堆（对象实例、数组，共享）、方法区（类信息、运行时常量池，共享）、虚拟机栈（方法调用，私有）、本地方法栈（Native 方法，私有）、程序计数器（字节码行号，私有）。堆是 GC 的主要区域。

### Q2：垃圾回收算法有哪些？

**标准回答**：
> 有三种：标记-清除（产生碎片）、复制（无碎片但浪费空间）、标记-整理（无碎片但移动成本高）。JVM 用分代收集：年轻代用复制算法（Eden + Survivor），老年代用标记-整理。常见收集器有 Serial、ParNew、Parallel Scavenge、Serial Old、Parallel Old、CMS、G1。

### Q3：双亲委派模型是什么？为什么需要？

**标准回答**：
> 双亲委派是类加载器先委派父加载器处理，父加载器处理不了才自己加载。好处是：1. 保证类的唯一性，父加载过的子加载器不再加载。2. 保证安全性，核心类不会被自定义类替换。比如用户自定义 java.lang.String 不会生效，因为会先由 Bootstrap ClassLoader 加载。

### Q4：Minor GC 和 Major GC 的区别？

**标准回答**：
> Minor GC 是年轻代的垃圾回收，频率高、停顿短。对象在 Eden 区分配，Minor GC 后存活的对象进入 Survivor 区，年龄够了进入老年代。Major GC 是老年代的垃圾回收，频率低、停顿长，通常伴随 Minor GC（Full GC）。G1 没有严格的新老年代区分。
