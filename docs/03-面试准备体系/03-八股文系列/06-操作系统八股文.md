## 一、进程与线程

### 1.1 进程 vs 线程

| 维度 | 进程 | 线程 |
|------|------|------|
| 定义 | 程序的执行实例 | 进程内的执行单元 |
| 资源 | 独立地址空间 | 共享进程资源 |
| 切换 | 开销大（切换页表、缓存） | 开销小（只切换寄存器） |
| 通信 | IPC（管道、消息队列等） | 直接读写共享内存 |
| 独立性 | 隔离，一个崩不影响其他 | 共享，一个崩可能影响其他 |
| 创建 | 慢 | 快 |

### 1.2 进程状态

```
┌────────┐     ┌────────┐     ┌────────┐
│  就绪  │────►│  运行  │────►│  阻塞  │
└────────┘     └────────┘     └────────┘
     ▲              │              │
     │              │              ▼
     └──────────────┘         ┌────────┐
                              │  终止  │
                              └────────┘
```

### 1.3 进程调度算法

| 算法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **FCFS** | 先来先服务 | 简单 | 长作业等太久 |
| **SJF** | 最短作业优先 | 平均等待最短 | 长作业可能饿死 |
| **时间片轮转** | 每个进程执行一个时间片 | 公平 | 时间片大小难定 |
| **优先级** | 按优先级调度 | 灵活 | 可能低优先级饿死 |
| **多级反馈队列** | 多队列 + 优先级 + 时间片 | 综合 | 复杂 |

## 二、线程同步

### 2.1 线程同步方式

| 方式 | 说明 | 适用场景 |
|------|------|---------|
| **互斥锁** | 独占资源 | 临界区 |
| **读写锁** | 读多写少 | 缓存 |
| **信号量** | 控制并发数 | 生产者消费者 |
| **条件变量** | 等待条件 | 线程协作 |
| **屏障** | 等待所有线程 | 并行计算 |

### 2.2 死锁

**四个必要条件**：
1. 互斥：资源只能被一个线程持有
2. 持有并等待：持有资源的同时请求其他资源
3. 不可抢占：资源不能被强制释放
4. 循环等待：线程之间循环等待资源

**解决死锁**：
- 预防：破坏四个条件之一
- 避免：银行家算法
- 检测：资源分配图 + 解除
- 忽略：重启

## 三、内存管理

### 3.1 内存分配方式

| 方式 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| **连续分配** | 连续一块内存 | 简单，访问快 | 内存碎片 |
| **分页** | 固定大小页 | 无外碎片 | 页表大 |
| **分段** | 按逻辑段分 | 符合编程习惯 | 外碎片 |
| **段页式** | 先分段再分页 | 两者优点 | 复杂 |

### 3.2 页面置换算法

| 算法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **FIFO** | 先来先换 | 简单 | Belady 异常 |
| **LRU** | 最久未用 |效果好 | 实现复杂 |
| **LFU** | 最少使用 | 效果好 | 实现复杂 |
| **Clock** | 近似 LRU | 折中 | — |

## 四、IO 模型

### 4.1 五种 IO 模型

| 模型 | 阻塞 | 非阻塞 | 多路复用 | 说明 |
|------|------|--------|---------|------|
| **阻塞 IO** | ✅ | | | 等数据就绪再返回 |
| **非阻塞 IO** | | ✅ | | 轮询查看是否就绪 |
| **IO 多路复用** | | | ✅ | select/poll/epoll |
| **信号驱动 IO** | | ✅ | | SIGIO 信号通知 |
| **异步 IO** | | | | 完全异步，内核完成 |

### 4.2 select/poll vs epoll

| 维度 | select | poll | epoll |
|------|--------|------|-------|
| 描述符数量 | 受限 FD_SETSIZE | 无限制 | 无限制 |
| 工作方式 | 遍历 | 遍历 | 回调 |
| 效率 | O(n) | O(n) | O(1) |
| 内存拷贝 | 每次都拷贝 | 每次都拷贝 | 只拷贝一次 |
| 适用 | 低并发 | 低并发 | 高并发 |

**epoll 的优势**：
- 不需要遍历所有 fd，用回调通知
- 只拷贝一次 fd 列表
- 支持边缘触发（ET）和水平触发（LT）

## 六、面试必问问题

### Q1：进程和线程的区别？

**标准回答**：
> 进程是程序执行实例，有独立地址空间，切换开销大。线程是进程内的执行单元，共享进程资源，切换开销小。进程隔离性好，一个崩不影响其他；线程共享资源，通信更简单但同步更复杂。

### Q2：什么是死锁？怎么解决？

**标准回答**：
> 死锁是四个条件同时满足：互斥、持有并等待、不可抢占、循环等待。解决方案有预防（破坏条件）、避免（银行家算法）、检测（资源分配图）、忽略（重启）。我项目中用 Redis 分布式锁避免死锁。

### Q3：select/poll/epoll 的区别？

**标准回答**：
> select 受 FD_SETSIZE 限制，每次都要遍历所有 fd，效率 O(n)。poll 没有数量限制但也是遍历。epoll 没有数量限制，用回调通知，效率 O(1)，适合高并发场景。Redis/Nginx 用的都是 epoll。
