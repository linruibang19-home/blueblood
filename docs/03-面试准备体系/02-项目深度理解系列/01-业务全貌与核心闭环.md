## 三、6 大业务域

| 业务域 | 前台 Tab | 后台分组 | 核心功能 |
|--------|---------|---------|---------|
| **社区协作** | 发现 | 社区管理 | 兴趣小组、帖子、评论、点赞/收藏、活动报名、私信 |
| **成长学习** | 成长 | 教育管理 | 课程、章节、学习进度、作业提交与批改、黑客松、AI 岗位 |

| **账户** | 我的·资料 | 平台运营·用户 | 资料/技能/实名认证/RBAC 角色/等级信誉日志 |
| **运营** | — | 工作台+平台运营 | 用户/认证审核、内容管理、系统配置、操作日志 |

## 五、核心业务流程（6 个）

### 流程 1：任务撮合 + 分阶段结算（平台核心 ⭐⭐⭐⭐⭐）

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    任务撮合 + 分阶段结算闭环                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│   雇主                    平台                    接单者                 │
│    │                       │                       │                     │
│    │ ①发布任务            │                       │                     │
│    │ (配多里程碑+金额)    │                       │                     │
│    │──────────────────────►│                       │                     │
│    │                       │ 初始化Redis库存         │                     │
│    │                       │────────────────────────────────────────────►│ │
│    │                       │                       │                     │
│    │                 ②接单(秒杀)                  │                     │
│    │◄──────────────────────────────────────────────────────────────────│ │
│    │ (Redis Lua预扣+DB兜底)                     │                     │
│    │                       │                       │                     │
│    │                 复制里程碑模板              │                     │
│    │                       │                       │                     │
│    │──────────────────────►│                       │                     │
│    │                       │                       │                     │
│    │                 ③提交里程碑                │                     │
│    │◄──────────────────────────────────────────────────────────────────│ │
│    │ (成果+附件)                                  │                     │
│    │                       │                       │                     │
│    │ ④审核里程碑          │                       │                     │
│    │──────────────────────►│                       │                     │
│    │ APPROVED/REJECTED     │                       │                     │
│    │                       │                       │                     │
│    │                 ⑤分阶段入账                  │                     │
│    │                       │────────────────────────────────────────────►│ │
│    │                       │ (幂等去重，不重复入账)  │                     │
│    │                       │                       │                     │
│    │                 ⑥订单settled                 │                     │
│    │                       │────────────────────────────────────────────►│ │
│    │                       │                       │                     │
│    │                       ▼                       ▼                     │
│    │              ┌────────────────┐      ┌────────────────┐          │
│    │              │ 雇主钱包(扣款)  │      │ 接单者钱包(入账)│          │
│    │              └────────────────┘      └────────────────┘          │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

**关键特点**：
- 分阶段结算：每个里程碑审核通过后，按该里程碑的金额入账
- 幂等去重：同一个里程碑只能入账一次
- 全部通过后订单 settled

### 流程 2：注册登录

```
发送验证码(Redis) → 注册(校验验证码) → 登录(JWT) → 后续请求带token
```

| 方式 | 凭证 | 验证码 |
|------|------|--------|
| 用户名注册 | username + password | 无需验证码 |
| 邮箱注册 | email + 验证码 | Redis 存 `verify:email:{email}` |
| 手机注册 | phone + 验证码 | Redis 存 `verify:phone:{phone}` |

### 流程 3：企业认证

```
企业注册(材料) → 提交认证申请(PENDING) → admin审核 → APPROVED(置enterprise)
```

- **个人用户**：学习、接单、社区
- **企业用户**：个人全部 + 发布任务/岗位/黑客松 + 雇主验收

### 流程 4：课程学习（线性进度模型）

```
选课 → 学第k章 → 标记完成 → completed_chapters=max(原,k) → 重算progress%
```

**特点**：只前进不回退（线性模型）

### 流程 5：钱包提现

```
申请提现(余额冻结) → 提现记录PENDING → admin审核 → 打款/驳回退款
```

- **提现**：原子扣余额（冻结）
- **驳回**：退还余额

### 流程 6：消息通知

```
触发通知 → 写入notification表 → 用户查未读数 → 标记已读
```

## 七、面试核心话术

### Q1：这个项目是做什么的？

**标准回答**：
> 蓝血菁英平台是一个面向 AI 技术人才的成长与变现一体化平台。用户可以学习课程提升技能，通过接任务交付成果赚钱，形成"学习→实战→结算"的闭环。

### Q2：项目有哪几端？你负责什么？

**标准回答**：
> 项目有 4 端：前台 H5、后台管理、微信小程序、后端 API，全部共用一套后端服务。我主要负责后端开发，涉及高并发、认证授权、核心业务逻辑等模块。

### Q3：核心业务流程是什么？

**标准回答**：
> 核心是任务撮合闭环：企业发布任务（配多里程碑各带金额）→ 用户接单 → 提交里程碑成果 → 雇主验收 → 按里程碑金额分阶段入账钱包 → 全部完成订单结算。

### Q4：三种用户角色分别能做什么？

**标准回答**：
> 个人用户可以学习、接单、社区、提现，但不能发布任务；企业用户认证后可以发布任务、岗位、黑客松；管理员负责后台全部运营管理。
