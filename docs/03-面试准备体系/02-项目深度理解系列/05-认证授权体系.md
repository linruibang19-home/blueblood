## 二、JWT 生成与验证

### 2.1 JWT 结构

**JWT = Header.Payload.Signature**

```
eyJhbGciOiJIUzI1NiJ9.eyJ1c2VySWQiOjEyMywidXNlcm5hbWUiOiJsaW4iLCJyb2xlIjoiVVNFUiIsImlhdCI6MTcwOTI4NDAwMH0.abc123...
├─ Header     ├─ Payload        ├─ Signature
```

**Header**：
```json
{
  "alg": "HS256",
  "typ": "JWT"
}
```

**Payload（负载）**：
```json
{
  "userId": 123,
  "username": "lin",
  "role": "USER",
  "iat": 1709284000,  // 签发时间
  "exp": 1709364000   // 过期时间
}
```

### 2.2 生成 Token

**项目代码**（[JwtUtil.java](../../server/blueblood-api/src/main/java/com/blueblood/api/security/JwtUtil.java)）：

```java
public String generate(Long userId, String username, String role) {
    Instant now = Instant.now();
    Instant exp = now.plusSeconds(props.getExpireSeconds());  // 7天过期
    
    return Jwts.builder()
            .issuer(props.getIssuer())
            .subject(String.valueOf(userId))
            .claim("userId", userId)
            .claim("username", username)
            .claim("role", role)
            .issuedAt(Date.from(now))
            .expiration(Date.from(exp))
            .signWith(key)  // 用 HMAC-SHA256 签名
            .compact();
}
```

### 2.3 验证 Token

```java
public boolean isValid(String token) {
    try {
        Claims claims = parse(token);
        return claims.getExpiration().after(Date.from(Instant.now()));
    } catch (JwtException | IllegalArgumentException e) {
        return false;  // 签名错误/过期/格式错误
    }
}

public LoginUser toLoginUser(String token) {
    Claims claims = parse(token);
    Long userId = claims.get("userId", Number.class).longValue();
    String username = claims.get("username", String.class);
    String role = claims.get("role", String.class);
    return new LoginUser(userId, username, role);
}
```

## 三、JWT 认证过滤器

### 3.1 过滤器链路

```
请求 → FilterChain
           │
           ▼
   ┌───────────────────┐
   │  CorsFilter        │ ← 处理跨域
   └─────────┬─────────┘
             │
             ▼
   ┌───────────────────┐

   │ Filter             │   存 SecurityContext
   └─────────┬─────────┘
             │
             ▼
   ┌───────────────────┐
   │ UsernamePassword   │ ← 表单登录（这里不用）
   │ Authentication... │
   └─────────┬─────────┘
             │
             ▼
   ┌───────────────────┐
   │  Controller       │ ← 业务处理
   └───────────────────┘
```

### 3.2 过滤器实现

**项目代码**（[JwtAuthenticationFilter.java](../../server/blueblood-api/src/main/java/com/blueblood/api/security/JwtAuthenticationFilter.java)）：

```java
@Component
@RequiredArgsConstructor
public class JwtAuthenticationFilter extends OncePerRequestFilter {
    
    private final JwtUtil jwtUtil;
    private final JwtProperties jwtProperties;
    
    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain) throws IOException, ServletException {
        // 1. 从请求头解析 token
        String token = resolveToken(request);
        
        // 2. 校验 token
        if (StringUtils.hasText(token) && jwtUtil.isValid(token)) {
            try {
                // 3. 还原登录用户
                LoginUser loginUser = jwtUtil.toLoginUser(token);
                
                // 4. 构造权限列表
                String role = loginUser.getRole() != null ? loginUser.getRole() : "USER";
                List<SimpleGrantedAuthority> authorities =
                        List.of(new SimpleGrantedAuthority("ROLE_" + role));
                
                // 5. 存入 SecurityContext
                UsernamePasswordAuthenticationToken authentication =
                        new UsernamePasswordAuthenticationToken(loginUser, null, authorities);
                authentication.setDetails(new WebAuthenticationDetailsSource().buildDetails(request));
                SecurityContextHolder.getContext().setAuthentication(authentication);
                
            } catch (Exception e) {
                SecurityContextHolder.clearContext();  // token 解析失败，清空
            }
        }
        
        // 6. 继续过滤器链
        filterChain.doFilter(request, response);
    }
    
    private String resolveToken(HttpServletRequest request) {
        String header = request.getHeader(jwtProperties.getHeader());
        if (StringUtils.hasText(header) && header.startsWith(jwtProperties.getTokenPrefix())) {
            return header.substring(jwtProperties.getTokenPrefix().length()).trim();
        }
        return null;
    }
}
```

## 四、Security 配置

### 4.1 SecurityFilterChain

**项目代码**（[SecurityConfig.java](../../server/blueblood-api/src/main/java/com/blueblood/api/config/SecurityConfig.java)）：

```java
@Bean
public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
    http
        // 1. 禁用 CSRF（前后端分离，token 在 header 里，不需要 CSRF）
        .csrf(AbstractHttpConfigurer::disable)
        
        // 2. 配置 CORS
        .cors(cors -> cors.configurationSource(corsConfigurationSource()))
        
        // 3. 无状态：不创建 Session
        .sessionManagement(sm -> sm.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
        
        // 4. 路径权限配置
        .authorizeHttpRequests(auth -> auth
            // 放行路径（无需登录）
            .requestMatchers(
                "/auth/login", 
                "/auth/register", 
                "/auth/wx-login",      // 微信登录
                "/auth/send-code",     // 发送验证码
                "/health",             // 健康检查
                "/uploads/**",         // 上传资源
                "/doc.html",           // Knife4j 文档
                "/swagger-ui/**",
                "/v3/api-docs/**"
            ).permitAll()
            
            // 其余需要认证
            .anyRequest().authenticated()
        )
        
        // 5. 异常处理（401/403）
        .exceptionHandling(eh -> eh
            .authenticationEntryPoint(authenticationEntryPoint)  // 401
            .accessDeniedHandler(accessDeniedHandler)            // 403
        )
        
        // 6. 把 JWT 过滤器加到 UsernamePasswordAuthenticationFilter 之前
        .addFilterBefore(jwtAuthenticationFilter, UsernamePasswordAuthenticationFilter.class);
    
    return http.build();
}
```

## 五、两层鉴权体系

### 5.1 第一层：RBAC（@PreAuthorize）

**作用**：控制能不能访问这个接口

```java
// 只有 ADMIN 才能访问
@PreAuthorize("hasRole('ADMIN')")
@PostMapping("/admin/user/disable")
public Result<?> disableUser(@RequestParam Long userId) { ... }

// 只有 ADMIN 才能访问
@PreAuthorize("hasRole('ADMIN')")
@PostMapping("/admin/task/approve")
public Result<?> approveTask(@RequestParam Long taskId) { ... }
```

**工作原理**：

```
1. 请求到达 Controller
       ↓
2. @PreAuthorize 拦截
       ↓
3. 获取 SecurityContext 里的 authorities
       ↓
4. 检查是否有 ROLE_ADMIN
       ├── 有 → 放行
       └── 无 → 403 Forbidden
```

### 5.2 第二层：业务级校验

**作用**：控制这个资源归不归你

**场景 1：数据归属校验**

```java
public TaskOrderVO orderDetail(Long orderId) {
    TaskOrder order = orderMapper.selectById(orderId);
    Long currentId = SecurityUtils.currentUserId();
    
    // 只有订单 owner / 任务 employer / admin 能查看
    boolean isOwner = order.getUserId().equals(currentId);
    boolean isAdmin = "ADMIN".equals(SecurityUtils.currentLoginUser().getRole());
    boolean isEmployer = task.getEmployerId().equals(currentId);
    
    if (!isOwner && !isAdmin && !isEmployer) {
        throw new BusinessException(ResultCode.FORBIDDEN, "无权查看该订单");
    }
    
    return toOrderVO(order);
}
```

**场景 2：user_type 校验**

```java
@Service
public class EnterpriseService {
    
    /** 确保当前用户是企业用户（发布任务前调用） */
    public void ensureEnterprise(Long userId) {
        User user = userMapper.selectById(userId);
        if (user == null || !"enterprise".equals(user.getUserType())) {
            throw new BusinessException(ResultCode.FORBIDDEN, "只有企业用户才能发布任务");
        }
    }
}

// Controller 里使用
@PostMapping("/task/publish")
public Result<?> publishTask(@RequestBody PublishTaskRequest req) {
    Long userId = SecurityUtils.currentUserId();
    enterpriseService.ensureEnterprise(userId);  // ← 校验 user_type
    return Result.success(taskPublishService.publish(req));
}
```

### 5.3 两层鉴权对比

| 层级 | 实现 | 管什么 |
|------|------|--------|
| **RBAC** | `@PreAuthorize` | 能不能访问这个接口 |
| **业务级** | Service 内校验 | 这个资源归不归你 |

## 六、为什么用无状态 JWT？

### 6.1 有状态 vs 无状态

| 维度 | 有状态（Session） | 无状态（JWT） |
|------|-----------------|---------------|
| 存储 | 服务端存 Session | 服务端不存储 |
| 扩展 | 需要 Session 同步 | 横向扩展简单 |
| 性能 | 占用服务端内存 | 无额外开销 |
| 撤销 | 立即生效 | 需要黑名单 |

### 6.2 JWT 的优缺点

**优点**：
1. 服务端无状态，横向扩展简单
2. 适合分布式系统
3. 性能好，无需查 Session

**缺点**：
1. Token 签发后无法撤销（除非加黑名单）
2. Token 体积比 Session ID 大
3. 敏感信息不能放 payload（可被解码，但不伪造）

### 6.3 项目的取舍

**为什么选 JWT？**
- 四端共用一套后端，无状态更合适
- 性能优先
- 实现简单

**为什么不选 Session？**
- 多节点需要 Session 同步
- 增加复杂度

## 七、面试必问问题

### Q1：JWT 怎么生成和验证？

**标准回答**：
> 用 JJWT 库生成。调用 `JwtUtil.generate(userId, username, role)` 签发 token，包含 userId/username/role/过期时间。验证时 `JwtUtil.isValid(token)` 校验签名和过期，`toLoginUser(token)` 还原登录用户。

### Q2：JWT 存在哪里？为什么？

**标准回答**：
> 存在 HTTP Header 的 Authorization 字段，格式是 `Bearer xxxxx`。不用 cookie 是因为前后端分离，不需要 CSRF 保护。存在 localStorage 每次请求手动带 header。

### Q3：无状态 vs 有状态的区别？

**标准回答**：
> 有状态 Session 存在服务端，分布式时需要同步，适合安全性要求高的场景。无状态 JWT 把用户信息放 token 里，服务端不存，横向扩展简单，但签发后无法撤销。

### Q4：两层鉴权是什么？

**标准回答**：
> 第一层是 @PreAuthorize 管功能权限，比如后台接口只有 ADMIN 能访问。第二层是 service 内校验管数据权限，比如这个订单归不归这个用户、user_type 是不是 enterprise。按场景分两层更清晰。
