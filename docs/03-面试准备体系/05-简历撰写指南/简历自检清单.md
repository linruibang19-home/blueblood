# 简历自检清单

> 写完简历后，逐条对照检查

---

## 一、基本信息

- [ ] 姓名、手机号、邮箱、城市都在
- [ ] 手机号格式正确（138-1234-5678）
- [ ] 邮箱不是 QQ 邮箱
- [ ] 求职意向明确（Java 后端开发）
- [ ] 没有无关信息（身份证、生日、照片）

---

## 二、教育背景

- [ ] 学校、专业、学历、时间正确
- [ ] GPA 还不错（专业前 20%）才写
- [ ] 没有硬凑的无关课程

---

## 三、项目经历

### 3.1 项目描述

- [ ] 一句话说清楚项目是干什么的
- [ ] 说了技术栈（前端用什么、后端用什么）
- [ ] 没有过度吹牛（一个实习生的项目不会改变世界）

### 3.2 主要工作（STAR 法则）

- [ ] 有具体的技术动作（不是"负责"、"参与"）
- [ ] 有量化结果（数字）
- [ ] 跟职位相关（后端开发多讲后端）
- [ ] 没有把所有功能都说成自己做的

### 3.3 常见问题检查

❌ **不好**：
```
负责后端开发
用 Redis 做缓存
参与数据库设计
```

✅ **好**：
```
设计并实现任务抢单系统，Redis Lua 预扣库存 + DB 原子 SQL 兜底，
解决超卖问题，单机 QPS 从 1000 提升到 10000。
```

### 3.4 检查清单

- [ ] 每个点都是具体的技术动作
- [ ] 每个技术点都能讲出"为什么"
- [ ] 没有语法错误/错别字
- [ ] 长度合适（一页纸，不要太满或太空）

---

## 四、技能清单

- [ ] 按掌握程度排序（熟悉 → 了解）
- [ ] 没有写"精通"
- [ ] 没有写无关技能（Word、Excel）
- [ ] 跟职位相关

---

## 五、其他

- [ ] 没有写自我评价（太主观）
- [ ] 荣誉奖励是真实的（能拿出证明）
- [ ] 校园经历跟技术相关（或者不写）

---

## 六、格式检查

- [ ] 一页纸（不超过两页）
- [ ] 字体统一（中文宋体、英文 Times New Roman）
- [ ] 字号统一（标题 14-16pt，正文 10-12pt）
- [ ] 行间距合适（不要太密或太松）
- [ ] 对齐正确（没有乱跑）
- [ ] 没有乱码

---

## 七、内容真实性检查

**每一条都要能回答这些问题**：
- [ ] 这个功能是你做的还是团队做的？
- [ ] 你是主力还是辅助？
- [ ] 遇到了什么困难？怎么解决的？
- [ ] 结果是怎么量化的？（数字从哪来）

**如果被问到这些，准备好答案**：
- [ ] 这个项目有多少人？你负责什么？
- [ ] 这个技术难点是谁解决的？
- [ ] 这个数字是怎么得出的？

---

## 八、面试前检查

每次面试前，检查：

- [ ] 简历最新版本
- [ ] 准备好转 PDF（避免格式乱）
- [ ] 能讲出简历上写的每个点
- [ ] 准备好项目演示（如果有）

---

## 九、常见问题修复

### 问题 1：项目描述太笼统

❌ 不好：
```
负责后端开发，用 Redis 做缓存
```

✅ 修复：
```
构建任务大厅缓存系统：互斥锁防击穿、空值标记防穿透、随机 TTL 防雪崩，
命中率从 0 提升到 80%，接口响应时间从 200ms 降到 50ms。
```

### 问题 2：没有量化结果

❌ 不好：
```
优化了数据库查询性能
```

✅ 修复：
```
优化任务列表查询：加联合索引、EXPLAIN 分析执行计划，
查询时间从 500ms 优化到 50ms，QPS 提升 10 倍。
```

### 问题 3：技术栈写太多

❌ 不好：
```
Java、Python、C、C++、Go、Rust、JavaScript、TypeScript、PHP...
```

✅ 修复：
```
Java（熟悉）、Python（了解）
Spring Boot、Spring MVC、Spring Security
MySQL（熟悉）、Redis（熟悉）
Docker、RabbitMQ、Linux
```

## 十、简历优化建议

### 优先级 1：量化结果

把"做了什么"改成"做了XX，YY提升了ZZ"：
- QPS 提升 X 倍
- 响应时间降低 Xms
- bug 修复 X 个
- 页面/模块重构涉及 X 个文件

### 优先级 2：突出亮点

挑 1-2 个最强的技术亮点重点写：
- 高并发秒杀
- 分布式限流
- 缓存三防
- 幂等设计

### 优先级 3：精简内容

删除：
- 无关的校园经历
- 太水的内容
- 重复的表达

---

## 十一、最终检查

打印出来（或放在手机里），读一遍：

1. **读出声来**：看看顺不顺口
2. **掐时间**：读一遍要多久（2-3 分钟正常）
3. **检查逻辑**：从项目介绍 → 技术亮点 → 成果，是不是一条线
4. **问自己**：如果我是面试官，想招什么样的人？

---

完成所有检查后，你的简历就可以用了！
