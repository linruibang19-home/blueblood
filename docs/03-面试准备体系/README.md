## ⏰ 学习时间规划

```
第一周：业务理解 + 并发体系（重点项目亮点）
第二周：八股文（MySQL + Redis + Spring）
第三周：项目深度 + 其他八股（计网/操作系统/JVM）
第四周：简历 + 面试技巧 + 算法刷题 + mock面试
```

**每天投入建议**：
- 在校期间：每天 2-3 小时
- 暑假/全职准备：每天 6-8 小时

## 📊 掌握程度自检

在每个阶段结束时，用这些问题自检：

### 阶段一自检
- [ ] 能画出业务架构图（4端 + 角色 + 业务域）
- [ ] 能讲清楚任务撮合闭环（6个步骤）
- [ ] 能说出每个角色能做什么

### 阶段二自检
- [ ] 能写出 Redis Lua 预扣的脚本逻辑
- [ ] 能解释为什么需要回补 Redis
- [ ] 能说出缓存三防分别是什么、怎么实现
- [ ] 能解释 @Transactional 为什么不管 Redis

### 阶段三自检
- [ ] 能脱稿讲 10 分钟项目介绍
- [ ] 能回答任何关于项目的追问
- [ ] 能说出项目中用到的每个技术为什么这么选

### 阶段四自检
- [ ] 能写出 MySQL 索引数据结构（B+树）
- [ ] 能解释 Redis 持久化（RDB/AOF）
- [ ] 能回答 Spring Bean 生命周期
- [ ] LeetCode hot 100 刷完或刷了 80%

## ⚠️ 重要提醒

1. **不要只看不练**：看完每个模块后，要讲给自己听或写笔记
2. **不要死记硬背**：理解原理比背答案更重要，面试官会追问
3. **不要只学八股文**：项目是核心，八股文是辅助，两者都要抓
4. **不要忽视算法**：大厂必考，每天 1-2 道保持手感
5. **不要闭门造车**：有机会就 mock 面试，练习表达
