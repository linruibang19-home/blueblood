# 01-学习路径与掌握标准

> 蓝血菁英平台 · 从零到面试成功的完整学习路径

---

## 一、什么是"真正掌握"？

掌握一个项目分为 **5 个层次**，从低到高：

| 层次 | 标准 | 你需要达到 |
|------|------|-----------|
| **L1 跑通** | 项目能跑，流程能走 | 能演示基本功能 |
| **L2 理解** | 知道每个模块干什么、怎么配合 | 能画架构图 |
| **L3 深入** | 理解技术原理、为什么这么选型 | 能回答追问 |
| **L4 复现** | 不看代码能画出核心流程图、关键类图 | 能讲清楚细节 |
| **L5 拓展** | 能改进设计、能迁移到其他场景 | 能说优化方向 |

**你的目标**：至少达到 **L3**，最好 **L4**。

---

## 二、学习阶段总览

```
阶段一：业务理解（3-5天）      → 达到 L2
阶段二：技术深入（7-10天）     → 达到 L3
阶段三：复现与表达（5-7天）    → 达到 L4
阶段四：冲刺准备（1-2周）      → 算法 + mock面试
```

## 五、阶段三：复现与表达（5-7 天）

### Day 13-14：准备自我介绍

**准备一个 10 分钟的项目介绍**：

```markdown
开头（1分钟）：
"这个项目叫蓝血菁英平台，是我实习期间做的 Java 重构版本..."

业务介绍（2分钟）：
"平台面向 AI 人才，做成长+变现一体化..."
"核心是任务撮合闭环..."

技术架构（3分钟）：
"后端 Spring Boot 3，前端 Vue3，MySQL+Redis..."
"四端共用一套 API..."

技术亮点（3分钟）：
"我主要负责高并发这块..."
"用 Redis Lua 做秒杀预扣..."
"用 try-catch 回补保证双写一致..."

个人贡献（1分钟）：
"我独立完成了任务模块的重构..."
"解决了 7 个真实 bug..."
```

**练习方式**：
1. 先照着念一遍
2. 然后不看书再讲一遍
3. 最后对镜子讲 3 遍

### Day 15-16：准备亮点讲解

准备每个亮点的"3 分钟版本"：

**亮点 1：Redis Lua 秒杀预扣**
- 背景：任务抢单是秒杀场景
- 问题：朴素"查→判→扣"会超卖
- 方案：Redis Lua 原子预扣 + DB 兜底
- 代码：画出流程图
- 一致性：try-catch 回补

**亮点 2：限流**
- 背景：验证码/登录接口需要防刷
- 方案：Redis + Lua 实现 @RateLimit AOP
- 代码：解释 Lua 脚本逻辑
- 追问：固定窗口的优缺点

**亮点 3：缓存三防**
- 防穿透：空值标记
- 防击穿：互斥锁
- 防雪崩：随机 TTL

## 八、遇到问题怎么办？

| 问题类型 | 解决方案 |
|---------|---------|
| 概念不懂 | 先看文档，再看视频，最后查博客 |
| 代码看不懂 | 先看整体结构，再逐行分析 |
| 讲不清楚 | 写下来，然后念，最后脱稿讲 |
| 算法不会 | 先想暴力解法，再想优化思路 |
| 心态崩了 | 休息一下，跟朋友聊聊，继续干 |

## 九、关键心态

1. **不要慌**：项目已经很完整了，你缺的是**把已有的东西讲清楚**
2. **不要背**：理解原理比背答案更重要，面试官会追问
3. **不要装**：说"AI 辅助做的"没问题，但要能讲清楚"我理解了 AI 做了什么"
4. **多练习**：讲 10 遍比想 10 遍更有效
5. **坚持住**：校招是一场持久战，保持节奏
