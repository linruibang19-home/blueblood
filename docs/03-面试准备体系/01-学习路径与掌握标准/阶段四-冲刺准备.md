## 三、Day 19-21：算法冲刺

### 3.1 刷题策略

**按这个顺序刷**：

| 优先级 | 类型 | 题目数量 | 推荐题目 |
|--------|------|---------|---------|
| P0 | 数组/链表 | 10-15 | 两数之和、合并两个有序链表、反转链表 |
| P0 | 二叉树/DFS/BFS | 10-15 | 二叉树层序遍历、路径总和、岛屿数量 |
| P1 | 排序/二分查找 | 5-10 | 二分查找、搜索旋转排序数组 |
| P1 | 动态规划 | 10-15 | 爬楼梯、背包问题、最长公共子序列 |
| P2 | 滑动窗口/双指针 | 5-10 | 无重复字符的最长子串、合并区间 |
| P2 | 栈/队列 | 5 | 有效的括号、栈实现队列 |

### 3.2 每天任务

**Day 19**：
- 刷 3 道数组/链表（hot 100 前 10）
- 复习今天做的题，理解思路

**Day 20**：
- 刷 3 道二叉树/DFS
- 复习今天做的题

**Day 21**：
- 刷 3 道动态规划
- 复习今天做的题

### 3.3 刷题技巧

1. **先想暴力解法**：不要一上来就想最优解
2. **分析复杂度**：时间复杂度、空间复杂度
3. **理解核心思路**：为什么要这么做？
4. **默写关键代码**：比如二分查找的边界处理

### 3.4 推荐资源

- LeetCode hot 100（必须刷完）
- 代码随想录（GitHub：programmercarl/labuladong）
- 灵神（B 站）

### 4.2 写简历的原则

1. **量化**：用数字说话，不要用"负责"、"参与"
2. **具体**：写清楚"做了什么"和"怎么做的"
3. **突出亮点**：项目最大的亮点要突出
4. **一页纸**：应届生一页就够了，不要超过两页

## 五、Day 25-28：八股文复习

### 5.1 复习顺序

```
Day 25 → MySQL（索引、事务、锁）
Day 26 → Redis（数据结构、持久化、缓存问题）
Day 27 → Spring（IoC、AOP、事务）
Day 28 → JVM（内存模型、GC、类加载）
```

### 5.2 每科重点

#### MySQL 重点

| 主题 | 必须掌握 | 了解 |
|------|---------|------|
| 索引 | B+ 树结构、聚簇/非聚簇索引、最左前缀原则 | 索引失效场景 |
| 事务 | 隔离级别、MVCC 原理 | 脏读/不可重复读/幻读 |
| 锁 | 行锁/表锁、意向锁、Next-Key Lock | 死锁原因 |
| SQL 优化 | EXPLAIN 分析、慢查询优化 | 索引设计原则 |

#### Redis 重点

| 主题 | 必须掌握 | 了解 |
|------|---------|------|
| 数据结构 | 5 种数据结构、底层实现 | 应用场景 |
| 持久化 | RDB、AOF、混合持久化 | 优缺点对比 |
| 缓存问题 | 穿透/击穿/雪崩/一致性问题 | 解决方案 |
| 集群 | 主从复制、哨兵、Cluster | 选型 |

#### Spring 重点

| 主题 | 必须掌握 | 了解 |
|------|---------|------|
| IoC | Bean 生命周期、作用域、循环依赖 | 启动流程 |
| AOP | 概念、代理方式（JDK/CGLIB）、应用场景 | 源码 |
| 事务 | @Transactional 属性、传播行为、失效场景 | 原理 |
| 安全 | Security 认证/授权流程 | 过滤器链 |

#### JVM 重点

| 主题 | 必须掌握 | 了解 |
|------|---------|------|
| 内存结构 | 堆/栈/方法区/本地方法栈/程序计数器 | 各区域作用 |
| GC | 回收算法、垃圾收集器（G1/ZGC） | CMS |
| 类加载 | 双亲委派模型、类加载过程 | 自定义类加载器 |
| 调优 | 常用参数、排查工具 | 实战 |

### 5.3 学习方法

1. **先看文档**：[03-八股文系列](../03-八股文系列/) 里的文档
2. **再搜博客**：针对不理解的概念搜博客
3. **最后实践**：如果有机会，写代码验证

## 六、Day 29-30：模拟面试

### 6.1 模拟内容

**技术面（45 分钟）**：
- 自我介绍（5 分钟）
- 项目介绍（10 分钟）
- 八股文（15 分钟）
- 算法题（15 分钟）

### 6.2 模拟方式

**方式 1：AI 模拟**
让 AI 扮演面试官，随机问问题。

**方式 2：朋友互面**
找也在准备面试的同学，互相当面试官。

**方式 3：录像**
对着摄像头讲，然后回放看哪里不好。

### 6.3 复盘

每次模拟后，整理：
- 哪些问题回答得好？
- 哪些问题回答得不好？
- 下次怎么改进？
