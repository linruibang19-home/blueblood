## Q1：线上 CPU 100% 怎么排查？

### 标准回答

> 排查步骤：
>
> **1. 定位进程**
> ```bash
> top  # 查看进程占用
> top -Hp <pid>  # 查看线程占用
> ```
>
> **2. 定位代码**
> ```bash
> jstack <pid>  # 导出线程快照
> # 找到 CPU 高的线程 tid，转 16 进制
> # 在 jstack 结果里找 nid=0x... 的线程
> ```
>
> **3. 分析**
> - RUNNABLE：死循环/正则/加密等 CPU 密集操作
> - WAITING：等待锁/IO
>
> **4. 常见原因**
> - 代码死循环
> - 正则回溯
> - 频繁 GC
> - 加密解密
>
> **项目中的场景**：
> - 任务大厅查询慢：看有没有没加索引
> - 限流检查慢：Lua 脚本问题？

>
> **项目中的场景**：

## Q3：线上 Redis 内存满了怎么办？

### 标准回答

> 排查步骤：
>
> **1. 查看内存使用**
> ```bash
> redis-cli info memory
> redis-cli --bigkeys  # 找大 key
> redis-cli --scan | head -100 | xargs redis-cli --bigkeys  # 扫描 key 大小
> ```
>
> **2. 分析 key**
> - 哪些 key 占内存多
> - key 的生命周期（有没有设过期时间）
>
> **3. 解决方案**
> - 加内存
> - 删无用数据
> - 改淘汰策略（`maxmemory-policy allkeys-lru`）
> - 分片存储
>
> **项目中的 key**：
> - `task:slots:{taskId}`：任务名额，TTL 不设
> - `task:list:*`：任务列表缓存，TTL 30-60s
> - `verify:*`：验证码，TTL 300s

## Q4：线上数据库连接池满了怎么办？

### 标准回答

> 排查步骤：
>
> **1. 查看连接数**
> ```sql
> SHOW STATUS LIKE 'Threads_connected';
> SHOW PROCESSLIST;  # 看有哪些连接
> ```
>
> **2. 看连接都在干什么**
> - 长时间不释放
> - 慢查询占用连接
> - 事务没提交
>
> **3. 常见原因**
> - 慢查询占用连接
> - 事务没提交/回滚
> - 连接泄漏（没 close）
>
> **4. 解决方案**
> - 加连接池大小
> - 优化慢查询
> - 检查代码有没有连接泄漏
> - 限流降级
>
> **项目中的配置**：
> ```yaml
> spring:
>   datasource:
>     hikari:
>       maximum-pool-size: 20
>       minimum-idle: 5
>       connection-timeout: 30000
> ```

## Q6：怎么设计一个缓存策略？

### 标准回答

> 缓存策略设计：
>
> **1. 缓存数据分类**
> | 类型 | 特点 | 策略 |
> |------|------|------|
> | 热点数据 | 频繁访问 | 永不过期，定期更新 |
> | 冷数据 | 偶尔访问 | TTL 短，及时淘汰 |
> | 配置数据 | 很少变化 | 长期缓存 |
>
> **2. 缓存更新策略**
> - Cache Aside（旁路缓存）：读时填充，写时删缓存
> - Read Through：缓存自动加载
> - Write Through：写缓存也写 DB
> - Write Behind：异步写 DB
>
> **3. 过期策略**
> - 固定 TTL：简单
> - 随机 TTL：防雪崩
> - 逻辑过期：不真过期，返回旧数据异步更新
>
> **项目中的策略**：
> - Cache Aside 模式
> - 任务列表随机 TTL
> - 幂等 token 固定 TTL

## Q7：怎么设计一个幂等接口？

### 标准回答

> 幂等接口设计：
>
> **1. 前端幂等**
> - 按钮置灰
> - 防重复点击
> - 重试时带唯一 ID
>
> **2. 后端幂等**
> | 方式 | 适用场景 |
> |------|---------|
> | 唯一索引 | 数据库层面防重 |
> | 状态机 | 状态流转 |
> | Token | 无唯一约束的写操作 |
> | 业务键去重 | 流水幂等 |
>
> **3. 实现**
> ```java
> // Token 幂等
> if (!idempotentService.consume("publish", token)) {
>     throw new BusinessException("请勿重复提交");
> }
>
> // 业务键去重
> WalletRecord existing = walletRecordMapper.selectOne(
>     new LambdaQueryWrapper<WalletRecord>()
>         .eq(WalletRecord::getBizType, "task")
>         .eq(WalletRecord::getBizId, milestoneId)
> );
> if (existing != null) return;  // 已处理
> ```
>
> **项目中的场景**：
> - 发布任务：Token 幂等
> - 结算入账：(biz_type, biz_id) 去重
