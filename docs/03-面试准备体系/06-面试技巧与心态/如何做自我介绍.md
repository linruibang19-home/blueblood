## 二、逐段模板

### 1. 开场（1 分钟）

```
"面试官好，我叫 XXX，今天来面试 Java 后端开发岗位。

很高兴能有机会参加这次面试。

我今天想分享一个我实习期间做的项目——蓝血菁英平台，是一个面向 AI 
技术人才的成长与变现一体化平台。"
```

**要点**：
- 简洁，不要太长
- 表明来意
- 引出项目

### 2. 项目介绍（3 分钟）

```
"这个平台解决的核心问题是：AI 人才怎么学习和变现？

平台围绕「成长 + 变现」双轮驱动，用户可以：
- 学习课程提升技能
- 接任务交付成果赚钱
- 参赛求职获取机会
- 钱包提现获得收益

平台有 4 端：前台 H5、后台管理、微信小程序、后端 API，全部共用 
一套后端服务。

核心业务是任务撮合闭环：企业发布任务（配多里程碑各带金额）→ 
用户接单 → 提交里程碑成果 → 雇主验收 → 按里程碑金额分阶段入账 
钱包 → 全部完成订单结算。"
```

**要点**：
- 一句话说清项目定位
- 举几个核心功能
- 重点讲任务撮合闭环

### 3. 技术架构（2 分钟）

```
"技术方面，后端用 Spring Boot 3 + MyBatis-Plus + MySQL 8 + Redis。

我主要负责后端开发，涉及的技术包括：
- JWT 无状态认证 + RBAC 双层鉴权
- Redis Lua 做秒杀预扣和限流
- 缓存三防（穿透/击穿/雪崩）
- 幂等 Token 防重复提交
- 钱包原子更新防丢失更新"
```

**要点**：
- 简单带过技术栈
- 重点说你的技术亮点

### 4. 技术亮点（3 分钟）— 重点！

```
"我重点做了高并发与稳定性体系这块。

**第一：任务抢单秒杀**

任务是抢单场景，名额有限，如果用朴素的「查→判→扣」会超卖。

我设计了一个双层架构：
- 第一层是 Redis Lua 预扣，Redis 单线程 + Lua 脚本保证「判断+扣减」
  原子执行，能挡瞬时高并发。
- 第二层是 DB 原子 SQL 兜底，`UPDATE task SET slots_left = slots_left - 1 
  WHERE id = ? AND slots_left > 0`，受影响行数为 0 说明已经满了。

还有一个关键问题：@Transactional 只回滚 DB，回滚不了 Redis。我用 
try-catch 回补：DB 失败时，手动调用 rollback 回补 Redis。虽然有短暂 
不一致，但不会超卖。

**第二：分布式限流**

验证码、登录接口需要防刷，我用 Redis + Lua 实现了一个 @RateLimit 
注解 + AOP 切面。

Lua 脚本逻辑是：INCR 计数 + 首次 EXPIRE 设置过期时间 + 超限判定。
支持按参数限流，也支持按 IP 限流。

**第三：缓存三防**

任务大厅列表走 Redis 缓存，但缓存有三大经典问题，我都做了防护：
- 防穿透：空结果也缓存（NULL_MARK，短 TTL）
- 防击穿：互斥锁，热点 key 过期时只让一个线程查 DB
- 防雪崩：随机 TTL（±30秒），避免同时失效"
```

**要点**：
- 挑 2-3 个最擅长的讲
- 讲清楚"问题 → 方案 → 为什么这么选"
- 能画流程图更好

### 5. 个人贡献（1 分钟）

```
"在项目中，我独立负责了任务模块的重构，包括接单、里程碑、钱包 
结算等核心功能。

开发过程中，我定位并修复了 7+ 个真实业务 bug，比如：
- 提现驳回不退款的资损 bug
- 并发超卖问题
- 进度计算卡死问题

最终，这个项目实现了 Docker Compose 5 服务一键部署，支撑四端共 
用一套后端 API。"
```

**要点**：
- 说你做了什么（不是团队做了什么）
- 量化成果
- 不要夸大

### 6. 结尾（1 分钟）

```
"以上就是我的项目介绍。

如果面试官有什么想深入了解的，我可以详细讲。

请问有什么想问的吗？"
```

## 三、练习方法

### 第一遍：照着念

把上面的内容复制到文档里，对着念一遍，掐时间。

### 第二遍：不看书

关掉文档，对着镜子讲一遍。注意：
- 语速不要太快
- 不要太生硬
- 适当停顿

### 第三遍：录音回放

用手机录音，回放听哪里不好：
- 有没有口头禅（"那个"、"嗯"、"就是"）
- 有没有逻辑不通的地方
- 语速是不是合适

### 第四遍：对着镜子或录像

看自己的表情和肢体语言：
- 眼神是不是自然（不要一直盯着一个地方）
- 表情是不是自然（不要僵硬）
- 手是不是乱动

### 第五遍：找人听

让朋友/同学/AI 当面试官，听他们听完后的反馈：
- 哪里听懂了？
- 哪里没听懂？
- 哪里感兴趣？

## 四、常见问题

### Q：项目介绍太长了怎么办？

**A**：挑重点讲，10 分钟已经够长了。核心是：
1. 项目是什么（1 分钟）
2. 你负责什么（3 分钟）
3. 技术亮点（5 分钟）
4. 个人贡献（1 分钟）

### Q：面试官打断我怎么办？

**A**：被打断说明他对某个点感兴趣，顺着往下讲。不要慌，不要强行回到你的稿子上。

### Q：讲乱了怎么办？

**A**：用"第一、第二、第三"来组织。每个技术亮点都用这个结构：
1. 背景/问题
2. 方案
3. 结果

---

## 五、最终检查

讲完一遍后，问自己：
- [ ] 能讲 10 分钟不停顿吗？
- [ ] 语速适中吗？
- [ ] 有没有口头禅？
- [ ] 逻辑清楚吗？
- [ ] 面试官能听懂吗？

如果都能做到，就可以了！
