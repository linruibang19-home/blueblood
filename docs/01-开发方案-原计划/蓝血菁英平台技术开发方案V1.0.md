# 蓝血菁英平台技术开发方案 V1.0

## 1. 项目定位

蓝血菁英平台是一个面向 AI 技术人才、学生、自由职业者和企业需求方的成长型任务撮合平台。

平台不是单纯社区，也不是单纯课程系统，而是围绕“超级个体”的成长、展示、连接、接单、交付和收益结算形成闭环。

核心业务闭环：

```text
用户注册/完善资料
-> 加入兴趣小组/社区互动
-> 学习课程/提交作业/参加黑客松
-> 获得积分/等级/信誉/技能标签
-> 进入任务大厅接单
-> 按里程碑提交交付物
-> 平台审核验收
-> 收益结算到账
-> 继续提升等级和接更高价值任务
```

## 2. 已确认技术路线

项目分四个阶段建设。

```text
第一阶段：Vue3 + Vite + TypeScript + Vant 复原前台 H5 原型
第二阶段：Spring Boot 3 + MyBatis-Plus + MySQL + Redis 建真实后端
第三阶段：RuoYi-Vue3 / Element Plus 建后台管理端
第四阶段：uni-app + Vue3 适配小程序端
```

当前优先目标是第一阶段：完整复原客户提供的 H5 前台原型，并为后续真实后端、后台管理端和小程序端预留架构边界。

## 3. Docker 跑动方案

后续项目应支持两种运行模式：

```text
本地 Docker 跑动：开发、联调、演示、交付给其他 AI/开发者。
服务器 Docker 跑动：测试环境、预生产环境、正式部署环境。
```

### 3.1 本地 Docker 建议

本地使用 Docker Compose 编排：

```text
blueblood-user-web      前台 H5 静态服务
blueblood-admin-web     后台管理静态服务
blueblood-api           Spring Boot API 服务
blueblood-mysql         MySQL 数据库
blueblood-redis         Redis 缓存
blueblood-nginx         统一入口网关
```

第一阶段只复原前台 H5 时，可以先只运行：

```text
blueblood-user-web
```

第二阶段后端加入后，再补齐：

```text
blueblood-api
blueblood-mysql
blueblood-redis
blueblood-nginx
```

### 3.2 服务器 Docker 建议

服务器端可以继续使用已有 Docker 中间件思路。

建议最终服务形态：

```text
blueblood-nginx         Nginx 统一入口
blueblood-user-web      用户 H5
blueblood-admin-web     后台管理端
blueblood-api           Spring Boot 后端
blueblood-redis         Redis
mysql                   服务器已有 MySQL 或 MySQL 容器
phpmyadmin              可保留用于数据库查看
blueblood-php           旧服务，可保留过渡，不作为新业务主线
```

推荐 Nginx 路由：

```text
/              -> 用户 H5
/admin/        -> 后台管理端
/api/          -> Spring Boot API
/uploads/      -> 文件资源
```

## 13. 开发约束

```text
页面内容必须由 Mock 数据驱动
组件命名清晰
路由命名清晰
不要把所有页面写在一个文件
不要直接复制原 HTML 形成不可维护代码
不要在 Vue 中大量使用原生 DOM 操作
所有金额字段按字符串或 number 显示，后端阶段使用 Decimal
所有状态值使用枚举
所有列表页面预留分页能力
所有上传功能第一阶段可做占位，第二阶段接真实接口
```

## 14. 总结

当前最合理的执行策略是先完成前台 H5 原型复原，把客户原型中的业务结构、页面结构和交互流程转为 Vue 工程。

第一阶段不要急于实现完整后端，但必须用 Mock 数据提前约束业务模型。这样后续 Spring Boot、后台管理和小程序适配都可以基于同一套业务对象继续建设。
